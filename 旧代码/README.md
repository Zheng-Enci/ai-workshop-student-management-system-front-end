# AI坊学生管理系统前端

人工智能创作坊学生管理系统前端项目，基于Vue3 + Element Plus构建的现代化学生管理平台。

## 项目概述

本项目是一个功能完整的学生管理系统前端，提供学生注册、登录、签到、个人信息管理等基础功能，以及强大的管理员控制台，支持学生信息管理、等级设置、签到统计等高级功能。

## 主要功能

### 学生功能
- 学生注册和登录
- 个人信息查看和修改
- 密码修改
- 分时段签到
- 个人签到次数查看
- 个人考勤记录查看（历史签到记录）

### 管理员功能
- 超级管理员身份验证
- 学生信息管理（查看、编辑、删除）
- 学生等级管理（社团成员、普通成员、核心成员、管理员）
- 学生签到次数统计
- 实时数据统计（总学生数、今日签到、本月签到）
- 按等级分类查看学生
- 学生管理员页面（查看所管理的所有学生信息）
- 学生补卡功能（管理员可为学生进行补卡操作）
- 查看所管理学生的考勤记录（历史签到记录）

### 数据看板
- 年级统计图表
- 专业统计图表
- 学生总数统计
- 优秀学生排行榜
- 月度/日度考勤统计

## 技术栈

- **前端框架**: Vue 3 (Composition API)
- **UI组件库**: Element Plus
- **路由管理**: Vue Router
- **状态管理**: Pinia
- **图表库**: ECharts
- **HTTP客户端**: Axios
- **构建工具**: Vue CLI

## 项目结构

```
src/
├── api/                    # API接口
│   ├── admin.js           # 管理员相关接口
│   └── user.js            # 用户相关接口
├── assets/                # 静态资源
├── components/            # 公共组件
├── router/                # 路由配置
├── stores/                # 状态管理
├── views/                 # 页面组件
│   ├── AdminPage.vue      # 超级管理员页面
│   ├── AttendancePage.vue # 签到页面
│   ├── DashboardPage.vue  # 数据看板页面
│   ├── LoginPage.vue      # 登录页面
│   ├── NavigationPage.vue # 导航页面
│   ├── ProfilePage.vue    # 个人资料页面
│   ├── RegisterPage.vue   # 注册页面
│   └── StudentManagerPage.vue # 学生管理员页面
├── App.vue                # 根组件
└── main.js                # 入口文件
```

## 快速开始

### 环境要求
- Node.js 16.0+
- npm 7.0+

### 安装依赖
```bash
npm install
```

### 开发环境运行
```bash
npm run serve
```

### 生产环境构建
```bash
npm run build
```

### 代码检查
```bash
npm run lint
```

## 页面说明

### 移动端页面
- **登录页面**: 学生登录入口，支持自动Token验证
- **注册页面**: 新学生注册，支持自动Token验证
- **导航页面**: 主要功能导航，包含签到、个人资料等
- **签到页面**: 分时段签到功能
- **个人资料页面**: 查看和修改个人信息，查看个人考勤记录
- **学生管理员页面**: 管理员查看所管理的所有学生信息，查看学生考勤记录

### 电脑端页面
- **数据看板页面**: 数据统计和图表展示
- **超级管理员页面**: 管理员控制台，需要特殊密码访问

## 特色功能

### 智能签到系统
- 分时段签到（上午、下午、晚上）
- 防重复签到机制
- 实时签到状态显示
- 管理员补卡功能（支持任意时间补卡）
- 考勤记录查看（学生可查看个人历史签到记录）
- 管理员可查看所管理学生的考勤记录

### 学生等级管理
- 四级等级系统（社团成员、普通成员、核心成员、管理员）
- 可视化等级标识
- 动态等级调整

### 数据统计分析
- 实时签到统计
- 学生分布统计
- 优秀学生排行
- 多维度数据图表

### 安全特性
- JWT Token身份验证
- 特殊密码管理员验证
- 自动Token过期处理
- 数据加密传输

## 部署说明

1. 构建生产版本：`npm run build`
2. 将`dist`目录部署到Web服务器
3. 配置后端API地址（修改`src/api/`下的baseURL）
4. 确保后端服务正常运行

## 注意事项

- 所有页面都支持响应式设计
- 数据看板和超级管理员页面仅在电脑端使用
- 超级管理员页面需要特殊密码才能访问
- 系统会自动验证Token有效性
- 签到功能有严格的时间段限制
