# 学生信息操作接口文档

## 文档概述

本文档描述了AI工作坊学生管理系统中所有与学生信息操作相关的API接口。学生信息操作模块提供了通过学生数据库表主键ID查询、获取学生信息的功能，支持灵活的信息查询和字段访问。

### 适用范围

- **目标用户**: 系统管理员、学生、第三方系统
- **权限级别**: 部分接口需要JWT Token认证，部分接口需要特殊密码认证，部分接口公开访问
- **使用场景**: 学生信息查询、字段值获取、信息验证、系统集成

### 安全说明

**重要**: 本文档中的敏感信息均使用占位符表示。实际使用时请：

1. 妥善保管JWT Token，不要泄露
2. 联系系统管理员获取真实的特殊密码
3. 不要在代码中硬编码敏感信息
4. 使用环境变量或安全的配置管理方式
5. 定期更换特殊密码和Token
6. 限制特殊密码的访问权限
7. 敏感字段（如password）不允许通过接口查询

## 基础信息

- **基础URL**: `http://<YOUR_SERVER_IP>:7001`
- **API版本**: v1
- **数据格式**: JSON
- **字符编码**: UTF-8
- **接口文档地址**: `http://<YOUR_SERVER_IP>:7001/doc.html`

## 通用响应格式

所有接口都使用统一的响应格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200表示成功，其他表示失败 |
| message | String | 响应消息 |
| data | Object | 响应数据，具体结构根据接口而定 |

### HTTP状态码说明

| 状态码 | 说明 | 常见场景 |
|--------|------|----------|
| 200 | 操作成功 | 请求正常处理 |
| 400 | 请求参数错误或业务逻辑错误 | 参数验证失败、业务规则违反、字段名不支持 |
| 401 | 认证失败 | Token无效或过期 |
| 500 | 服务器内部错误 | 数据库连接失败、系统异常 |

## 认证方式

### JWT Token认证

部分接口需要JWT Token认证，通过Query参数传递：

```
?token=<YOUR_JWT_TOKEN>
```

**注意**: JWT Token属于敏感信息，请妥善保管，不要泄露。

### 特殊密码认证

部分接口需要特殊密码认证，通过Query参数传递：

```
?special-password=<YOUR_SPECIAL_PASSWORD>
```

**注意**: 特殊密码属于敏感信息，请妥善保管，不要泄露。

## 1. 获取当前学生信息

### 1.1 获取学生信息

**接口描述**: 通过JWT Token获取当前登录学生的详细信息，包括学号、姓名、性别、联系方式、院系、年级、专业、班级等信息。

**接口地址**: `GET /api/v1/students/profile`

**请求方法**: `GET`

**访问控制**:
- 该接口需要JWT Token认证
- 该接口无需特殊密码

**请求参数** (Query Parameters):

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| token | String | 是 | JWT token |

**请求示例**:

```http
GET /api/v1/students/profile?token=<YOUR_JWT_TOKEN> HTTP/1.1
Host: <YOUR_SERVER_IP>:7001
```

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "获取学生信息成功",
  "data": {
    "studentId": 2021001001,
    "name": "张三",
    "gender": "男",
    "phoneNumber": "13812345678",
    "college": "计算机学院",
    "grade": 3,
    "major": "软件工程",
    "classNum": 1
  }
}
```

失败响应 (401) - Token无效:
```json
{
  "code": 401,
  "message": "Token 无效",
  "data": null
}
```

失败响应 (400):
```json
{
  "code": 400,
  "message": "获取学生信息失败",
  "data": null
}
```

**业务规则**:
- 需要有效的JWT Token进行身份验证
- 返回当前登录学生的详细信息
- 不包含密码等敏感信息
- 不包含学生数据库表主键ID

**注意事项**:
- Token必须有效且未过期
- 该接口只能获取当前登录学生自己的信息
- 如需获取其他学生信息，需要使用其他接口

## 2. 获取所有学生信息

### 2.1 通过特殊密码获取所有学生信息

**接口描述**: 通过特殊密码获取系统中所有学生的完整信息，包括学生表主键ID、学号、姓名、性别、联系方式、院系、年级、专业、班级等信息。

**接口地址**: `GET /api/v1/students/all-with-special-password`

**请求方法**: `GET`

**访问控制**:
- 该接口需要特殊密码认证
- 该接口无需Token认证
- 通过Query参数传递特殊密码

**请求参数** (Query Parameters):

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| special-password | String | 是 | 特殊密码（请联系系统管理员获取） |

**请求示例**:

```http
GET /api/v1/students/all-with-special-password?special-password=<YOUR_SPECIAL_PASSWORD> HTTP/1.1
Host: <YOUR_SERVER_IP>:7001
```

**响应示例**:

成功响应 (200):

```json
{
  "code": 200,
  "message": "获取所有学生信息成功",
  "data": [
    {
      "id": 1,

      "name": "张三",
      "gender": "男",
      "phoneNumber": "13812345678",
      "college": "计算机学院",
      "grade": 3,
      "major": "软件工程",
      "classNum": 1
    },
    {
      "id": 2,
      "studentId": 2021001002,
      "name": "李四",
      "gender": "女",
      "phoneNumber": "13812345679",
      "college": "计算机学院",
      "grade": 2,
      "major": "计算机科学与技术",
      "classNum": 2
    }
  ]
}
```

失败响应 (400) - 特殊密码错误:
```json
{
  "code": 400,
  "message": "特殊密码错误",
  "data": null
}
```

**响应字段说明** (StudentInfoWithIdVO):

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 学生表主键ID（student_info表的主键id） |
| studentId | Long | 学号 |
| name | String | 姓名 |
| gender | String | 性别 |
| phoneNumber | String | 手机号 |
| college | String | 学院 |
| grade | Integer | 年级 |
| major | String | 专业 |
| classNum | Integer | 班级 |

**业务规则**:
- 需要提供正确的特殊密码
- 返回系统中所有学生的详细信息
- 包含学生表主键ID（`id`字段），便于后续操作
- 不包含密码等敏感信息
- 用于管理员查看所有学生信息

**注意事项**:
- 特殊密码必须正确，否则返回400错误
- 该接口返回所有学生信息，数据量可能较大
- 包含学生表主键ID，可用于后续通过ID查询学生信息
- 建议在生产环境中实现分页功能

## 3. 获取学生数据库ID

### 3.1 获取学生数据库表主键ID

**接口描述**: 通过JWT Token获取当前登录学生在数据库中的唯一ID（学生表主键ID）。

**接口地址**: `GET /api/v1/students/get-student-database-table-id`

**请求方法**: `GET`

**访问控制**:
- 该接口需要JWT Token认证
- 该接口无需特殊密码

**请求参数** (Query Parameters):

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| token | String | 是 | JWT token |

**请求示例**:

```http
GET /api/v1/students/get-student-database-table-id?token=<YOUR_JWT_TOKEN> HTTP/1.1
Host: <YOUR_SERVER_IP>:7001
```

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "获取学生ID成功",
  "data": 123
}
```

失败响应 (401) - Token无效:
```json
{
  "code": 401,
  "message": "Token 无效",
  "data": null
}
```

失败响应 (400) - Token解析错误:
```json
{
  "code": 400,
  "message": "获取学生ID失败: Token解析错误",
  "data": null
}
```

**业务规则**:
- 需要有效的JWT Token进行身份验证
- 返回当前登录学生在数据库中的主键ID
- 用于系统内部ID查询和关联操作
- 不返回学号，只返回数据库主键ID

**注意事项**:
- Token必须有效且未过期
- 返回的是学生表主键ID（Integer类型），不是学号
- 该ID可用于后续通过ID查询学生信息的接口
- 数据安全：只返回数据库主键ID，不暴露其他敏感信息

## 4. 根据数据库ID获取学生姓名

### 4.1 根据数据库主键ID获取学生姓名

**接口描述**: 通过学生数据库表主键ID获取学生姓名。该接口为公开接口，无需认证。

**接口地址**: `GET /api/v1/students/get-student-name-by-info-id`

**请求方法**: `GET`

**访问控制**:
- 该接口无需Token认证
- 该接口无需特殊密码
- 公开访问接口

**请求参数** (Query Parameters):

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| student-info-id | Integer | 是 | 数据库主键ID |

**请求示例**:

```http
GET /api/v1/students/get-student-name-by-info-id?student-info-id=1 HTTP/1.1
Host: <YOUR_SERVER_IP>:7001
```

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "获取学生姓名成功",
  "data": {
    "name": "张三"
  }
}
```

失败响应 (400) - 学生不存在:
```json
{
  "code": 400,
  "message": "该学生不存在",
  "data": null
}
```

**响应字段说明** (StudentNameByStudentInfoIdVO):

| 字段名 | 类型 | 说明 |
|--------|------|------|
| name | String | 学生姓名 |

**业务规则**:
- 根据数据库主键ID查询学生姓名
- 返回学生的姓名信息
- 无需Token验证，公开接口
- 用于通过数据库ID快速查询学生姓名

**注意事项**:
- `student-info-id` 是学生表的主键ID，不是学号
- 该接口只返回姓名字段，不暴露其他敏感信息
- 轻量级接口，响应速度快
- 适用于需要根据ID快速获取学生姓名的场景

## 5. 根据ID和字段名获取可公开字段值

### 5.1 根据ID和字段名获取学生可公开字段值

**接口描述**: 通过学生数据库表主键ID和字段名，获取对应可公开字段的值。该接口支持动态查询学生信息中的可公开字段，如姓名、性别、学院、年级、专业等。该接口为公开接口，无需认证。

**接口地址**: `GET /api/v1/students/public-field-value-by-id`

**请求方法**: `GET`

**访问控制**:
- 该接口无需Token认证
- 该接口无需特殊密码
- 公开访问接口

**请求参数** (Query Parameters):

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| student-info-id | Long | 是 | 学生数据库表主键ID |
| field-name | String | 是 | 字段名（支持的字段：name, gender, college, grade, major） |

**支持的字段列表**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| name | String | 姓名 |
| gender | String | 性别 |
| college | String | 学院 |
| grade | Integer | 年级 |
| major | String | 专业 |

**请求示例**:

```http
GET /api/v1/students/public-field-value-by-id?student-info-id=1&field-name=name HTTP/1.1
Host: <YOUR_SERVER_IP>:7001
```

**响应示例**:

成功响应 (200) - 查询姓名字段:
```json
{
  "code": 200,
  "message": "获取成功",
  "data": "张三"
}
```

成功响应 (200) - 查询年级字段:
```http
GET /api/v1/students/public-field-value-by-id?student-info-id=1&field-name=grade HTTP/1.1
```

```json
{
  "code": 200,
  "message": "获取成功",
  "data": "3"
}
```

失败响应 (400) - 字段名不支持:
```json
{
  "code": 400,
  "message": "不支持的字段: phoneNumber，支持的字段有: name, gender, college, grade, major",
  "data": null
}
```

失败响应 (400) - 字段名为空:
```json
{
  "code": 400,
  "message": "字段名不能为空",
  "data": null
}
```

失败响应 (400) - 学生不存在:
```json
{
  "code": 400,
  "message": "该学生不存在",
  "data": null
}
```

**业务规则**:
- 根据学生数据库表主键ID和字段名查询对应字段值
- 只支持查询可公开字段，敏感字段（如password）不允许查询
- 所有字段值统一返回String类型，前端根据字段类型自行转换
- 字段名区分大小写，使用小写
- 无需Token验证，公开接口

**注意事项**:
- `student-info-id` 是学生表的主键ID，不是学号
- `field-name` 必须是支持的字段之一，否则返回400错误
- 支持的字段：name, gender, college, grade, major
- 不支持的字段（如password、phoneNumber等）会返回错误
- 该接口适用于需要动态查询学生特定字段值的场景
- 所有字段值统一返回String类型，数字类型字段（如grade）也会转换为String返回

**安全说明**:
- 敏感字段（如password）不在支持的字段列表中，无法通过该接口查询
- 字段名通过硬编码白名单验证，防止SQL注入等安全问题
- 只返回可公开的字段值，不暴露敏感信息

## 注意事项

1. **Token安全**:
   - JWT Token必须妥善保管，不要泄露
   - Token有过期时间，过期后需要重新登录
   - 不要在代码中硬编码Token

2. **特殊密码安全**:
   - 特殊密码必须妥善保管，不要泄露
   - 不要在代码中硬编码特殊密码
   - 建议使用环境变量或配置中心管理

3. **参数验证**:
   - 所有必填参数必须提供
   - 参数格式必须符合要求
   - 字段名必须在支持的列表中

4. **数据一致性**:
   - 学生数据库表主键ID是唯一标识
   - 学号和数据库表主键ID是不同的概念
   - 通过ID查询的信息是实时数据

5. **性能优化**:
   - 获取所有学生信息接口数据量可能较大，建议实现分页
   - 字段值查询接口响应速度快，适合高频访问
   - 建议合理使用缓存优化性能

6. **错误处理**:
   - 所有错误都会返回统一的错误格式
   - 错误信息会明确说明失败原因
   - 建议根据错误码进行相应的错误处理

7. **敏感信息保护**:
   - 密码等敏感字段不允许通过接口查询
   - 只返回可公开的字段信息
   - 遵循最小权限原则

8. **字段名规范**:
   - 字段名使用小写
   - 字段名区分大小写
   - 只支持预定义的可公开字段
