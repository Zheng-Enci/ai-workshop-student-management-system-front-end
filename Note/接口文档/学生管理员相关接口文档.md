# 学生管理员相关接口文档

## 模块概述

学生管理员相关接口提供了学生管理员管理学生的功能，包括查询管理员管理的学生信息及其统计数据。该模块支持管理员查看自己管理的学生的基本信息、签到次数和总积分等详细统计数据。

## 接口列表

### 1. 获取管理的学生列表（含统计）

**接口地址**: `GET /api/v1/student-management/managed-students-with-stats`

**接口描述**: 通过JWT token获取当前用户（管理员）管理的所有学生的详细信息，包含签到次数和总积分统计。

**请求头**:
```
Accept: application/json
```

**请求参数**:

| 参数名 | 类型 | 位置 | 必填 | 说明 |
|--------|------|------|------|------|
| token | String | Query | 是 | JWT token，用于验证身份和获取管理员ID |

**请求示例**:
```

```

**响应模型**: `StudentInfoWithStatsVO`

| 字段 | 类型 | 说明 |

| id | Long | 学生数据库主键ID |

**业务规则**:
- 需要有效的JWT token进行身份验证
- 只有管理员权限的学生（等级3）才能查询管理的学生信息
- 返回该管理员管理的所有学生的详细信息
- 包含学生的基本信息和统计数据（签到次数和总积分）
- 统计数据基于学生的历史考勤记录计算

**安全特性**:
- Token验证：确保只有已登录的管理员才能访问
- 权限控制：只有管理员等级的学生才能查看管理的学生信息
- 数据安全：只返回管理员权限范围内的学生信息

**统计计算规则**:
- 签到次数：统计学生所有历史签到记录的总数
- 总积分：按照签到次数 × 10的规则计算（每次签到获得10积分）
- 统计数据实时计算，确保数据的准确性

## 业务流程

### 获取管理学生信息流程

1. **身份验证**: 
   - 客户端提供JWT token
   - 系统验证token的有效性和格式
   - 提取token中的学生信息ID

2. **权限验证**:
   - 验证该学生是否为管理员（等级3）
   - 如果不是管理员，拒绝访问

3. **数据查询**:
   - 根据管理员ID查询student_management表获取管理的所有学生ID
   - 根据学生ID查询student表获取基本信息
   - 查询attendance表统计每个学生的签到次数
   - 根据签到次数计算总积分

4. **数据组装**:
   - 将基本信息和统计数据组装成StudentInfoWithStatsVO对象
   - 返回完整的响应数据

## 数据库设计

### 学生管理关系表 (student_management)

```sql
CREATE TABLE student_management (
    id INT PRIMARY KEY AUTO_INCREMENT,
    admin_student_info_id INT NOT NULL,
    managed_student_info_id INT UNIQUE NOT NULL,
    FOREIGN KEY (admin_student_info_id) REFERENCES student_info(id),
    FOREIGN KEY (managed_student_info_id) REFERENCES student_info(id),
    INDEX idx_admin_id (admin_student_info_id),
    INDEX idx_managed_id (managed_student_info_id)
);
```

### 关联查询优化

为了提高查询性能，建议创建以下索引：

```sql
-- 管理员ID索引，用于快速查找管理的学生
CREATE INDEX idx_admin_student_info_id ON student_management(admin_student_info_id);

-- 被管理学生ID索引，确保唯一性查询性能
CREATE INDEX idx_managed_student_info_id ON student_management(managed_student_info_id);

-- 学生表学号索引，用于基本信息查询
CREATE INDEX idx_student_id ON student_info(student_id);

-- 考勤表学生ID索引，用于统计签到次数
CREATE INDEX idx_attendance_student_info_id ON attendance(student_info_id);
```

## 错误处理

### 常见错误码

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| 401 | Token无效或过期 | 重新登录获取新Token |
| 400 | 学生不是管理员 | 确保该学生等级为3（管理员） |
| 400 | 该学生没有管理关系 | 确保该管理员有管理的学生 |
| 500 | 服务器内部错误 | 联系系统管理员 |

### 错误响应示例

**Token无效错误**:
```json
{
  "code": 401,
  "message": "Token 无效",
  "data": null
}
```

**权限不足错误**:
```json
{
  "code": 400,
  "message": "该学生不是管理员，无法查看管理的学生",
  "data": null
}
```

**没有管理学生错误**:
```json
{
  "code": 200,
  "message": "获取管理学生信息成功",
  "data": []
}
```

## 性能优化

### 数据库优化

1. **索引优化**:
   - 在student_management表的admin_student_info_id字段上创建索引
   - 在student_info表的id字段上创建主键索引
   - 在attendance表的student_info_id字段上创建索引

2. **查询优化**:
   - 使用JOIN查询减少数据库访问次数
   - 使用聚合函数提高统计查询效率
   - 考虑添加缓存机制提高查询性能

### 缓存策略

1. **统计数据缓存**:
   - 考虑缓存学生的签到次数和总积分
   - 设置合理的缓存过期时间（如1小时）
   - 在学生签到时更新相关缓存

2. **管理关系缓存**:
   - 缓存管理员管理的学生关系
   - 在分配或取消管理关系时更新缓存

## 安全考虑

### 权限控制

1. **Token验证**:
   - 严格验证JWT token的有效性
   - 检查token是否过期
   - 验证token签名

2. **等级验证**:
   - 确保只有等级3（管理员）的学生才能访问
   - 在每次请求时验证学生等级
   - 防止权限提升攻击

### 数据安全

1. **数据过滤**:
   - 只返回管理员权限范围内的学生信息
   - 不包含密码等敏感信息
   - 限制返回数据的字段

2. **审计日志**:
   - 记录所有管理相关的操作
   - 包括操作时间、操作人、操作内容
   - 便于安全审计和问题排查
