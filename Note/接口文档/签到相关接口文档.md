# 考勤管理接口文档

## 文档概述

本文档描述了AI工作坊学生管理系统中所有与考勤管理相关的API接口。考勤管理模块提供了学生签到、考勤统计、补卡管理、排名查询等核心功能，支持对学生考勤进行全面的管理和分析。

### 适用范围

- **目标用户**: 系统管理员、学生
- **权限级别**: 部分接口需要JWT Token认证，部分接口需要特殊密码认证，部分接口公开访问
- **使用场景**: 学生签到、考勤统计、补卡操作、排名查询、验证码获取

### 安全说明

**重要**: 本文档中的敏感信息均使用占位符表示。实际使用时请：

1. 妥善保管JWT Token，不要泄露
2. 联系系统管理员获取真实的特殊密码
3. 不要在代码中硬编码敏感信息
4. 使用环境变量或安全的配置管理方式
5. 定期更换特殊密码和Token
6. 限制特殊密码的访问权限
7. 验证码接口仅限特定IP访问，请勿公开IP白名单配置

## 基础信息

- **基础URL**: `http://<YOUR_SERVER_IP>:7001`
- **API版本**: v1
- **数据格式**: JSON
- **字符编码**: UTF-8
- **接口文档地址**: `http://<YOUR_SERVER_IP>:7001/doc.html`

## 通用响应格式

所有接口都使用统一的响应格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200表示成功，其他表示失败 |
| message | String | 响应消息 |
| data | Object | 响应数据，具体结构根据接口而定 |

### HTTP状态码说明

| 状态码 | 说明 | 常见场景 |
|--------|------|----------|
| 200 | 操作成功 | 请求正常处理 |
| 400 | 请求参数错误或业务逻辑错误 | 参数验证失败、业务规则违反、验证码错误 |
| 401 | 认证失败 | Token无效或过期 |
| 403 | 禁止访问 | IP不在白名单内、权限不足 |
| 500 | 服务器内部错误 | 数据库连接失败、系统异常 |

## 认证方式

### JWT Token认证

部分接口需要JWT Token认证，通过Query参数传递：

```
?token=<YOUR_JWT_TOKEN>
```

**注意**: JWT Token属于敏感信息，请妥善保管，不要泄露。

### 特殊密码认证

部分接口需要特殊密码认证，通过请求体传递：

```json

```

**注意**: 特殊密码属于敏感信息，请妥善保管，不要泄露。

### IP白名单认证

验证码接口需要IP白名单认证，只有配置在白名单中的IP才能访问。

## 1. 签到操作接口

### 1.1 学生签到

**接口描述**: 学生在指定时间段内进行签到，需要提供有效的JWT token和当前有效的签到验证码。系统会根据当前时间自动识别签到时间段，并防止重复签到。

**接口地址**: `POST /api/v1/attendance`

**请求方法**: `POST`

**访问控制**:
- 该接口需要JWT Token认证
- 该接口需要签到验证码验证
- 验证码通过IP白名单保护的接口获取

**请求参数** (Query Parameters):

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| token | String | 是 | JWT token |
| verificationCode | String | 是 | 签到验证码（6位数字），通过获取验证码接口获取 |

**签到时间段**:

| 时间段 | 时间范围 | 说明 |
|--------|----------|------|
| 上午 | 08:00 - 11:00 | 上午签到时间段 |
| 下午 | 14:00 - 17:00 | 下午签到时间段 |
| 晚上 | 19:00 - 22:00 | 晚上签到时间段 |

**请求示例**:

```http
POST /api/v1/attendance?token=<YOUR_JWT_TOKEN>&verificationCode=123456 HTTP/1.1
Host: <YOUR_SERVER_IP>:7001
Content-Type: application/json
```

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "签到成功",
  "data": {
    "message": "签到成功"
  }
}
```

失败响应 (400) - 验证码错误:
```json
{
  "code": 400,
  "message": "验证码错误或已过期",
  "data": null
}
```

失败响应 (400) - 不在签到时间段:
```json
{
  "code": 400,
  "message": "当前时间不在签到时间段内",
  "data": null
}
```

失败响应 (400) - 重复签到:
```json
{
  "code": 400,
  "message": "您已于本期签到，请勿重复签到",
  "data": null
}
```

**业务规则**:
- 每个时间段只能签到一次
- 使用Redis缓存防止重复签到，缓存Key格式：`attendance:{studentId}:{period}:{date}`
- 签到记录永久保存到数据库
- 超出时间段无法签到
- 系统自动识别当前时间段
- 验证码每16秒自动刷新

**注意事项**:
- Token必须有效且未过期
- 验证码必须通过IP白名单保护的接口获取
- 验证码有效期为16秒，过期后需要重新获取
- 同一时间段内重复签到会返回错误
- 建议在签到前先获取最新的验证码

## 5. 排名查询接口

### 5.1 获取签到次数排名前N名

**接口描述**: 根据签到次数降序排列，返回前N名学生的学生表主键ID和签到次数。该接口为公开接口，无需认证。

**接口地址**: `GET /api/v1/attendance/top-ranking`

**请求方法**: `GET`

**访问控制**:
- 该接口无需Token认证
- 该接口无需特殊密码
- 公开访问接口

**请求参数** (Query Parameters):

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| top | Integer | 否 | 10 | 排名数量，最小值为1，无上限 |

**请求示例**:

```http
GET /api/v1/attendance/top-ranking?top=10 HTTP/1.1
Host: <YOUR_SERVER_IP>:7001
```

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "获取成功",
  "data": [
    {
      "studentInfoId": 1,
      "attendanceCount": 50
    },
    {
      "studentInfoId": 5,
      "attendanceCount": 45
    },
    {
      "studentInfoId": 3,
      "attendanceCount": 40
    }
  ]
}
```

失败响应 (400) - 参数验证失败:
```json
{
  "code": 400,
  "message": "排名数量必须大于等于1",
  "data": null
}
```

**响应字段说明** (StudentInfoIdAndAttendanceCountVO):

| 字段名 | 类型 | 说明 |
|--------|------|------|
| studentInfoId | Long | 学生表主键ID（student_info表的主键id） |
| attendanceCount | Long | 签到次数（attendance表中的记录总数） |

**业务规则**:
- 签到次数 = 该学生在 `attendance` 表中的记录总数
- 计算公式：`COUNT(id)`
- 如果学生没有签到记录，不包含在结果中
- 结果按签到次数降序排列（从高到低）
- 如果签到次数相同，排序可能不确定（取决于数据库实现）

**注意事项**:
- `top` 参数范围：最小值为1，无上限，小于1会返回400错误
- 如果不传 `top` 参数，默认返回前10名
- `studentInfoId` 是学生表的主键ID，不是学号
- 该接口只返回学生ID和签到次数，不包含学生其他信息
- 如需获取学生详细信息，需要根据 `studentInfoId` 调用学生信息查询接口

## 6. 验证码接口

### 6.1 获取验证码

**接口描述**: 获取当前有效的六位数验证码，每16秒自动刷新。该接口为公开接口，无需认证，但是只有特定IP才能访问。

**接口地址**: `GET /api/v1/attendance/verification-code`

**请求方法**: `GET`

**访问控制**:
- 该接口无需Token认证
- 该接口无需特殊密码
- 该接口需要IP白名单认证
- 只有配置在白名单中的IP才能访问

**请求参数**: 无

**请求示例**:

```http
GET /api/v1/attendance/verification-code HTTP/1.1
Host: <YOUR_SERVER_IP>:7001
```

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "获取验证码成功",
  "data": "123456"
}
```

失败响应 (403) - IP不在白名单:
```json
{
  "code": 403,
  "message": "IP地址不允许访问该接口，您的IP: <CLIENT_IP>",
  "data": null
}
```

失败响应 (403) - IP白名单未配置:
```json
{
  "code": 403,
  "message": "IP白名单未配置",
  "data": null
}
```

**业务规则**:
- 验证码为6位数字
- 验证码每16秒自动刷新
- 只有配置在白名单中的IP才能访问
- 验证码用于学生签到接口

**注意事项**:
- IP白名单配置在服务器端，不在本文档中暴露
- 验证码有效期16秒，过期后需要重新获取
- 建议在签到前先获取最新的验证码
- 该接口仅限特定IP访问，请勿公开IP白名单配置

## 注意事项

1. **Token安全**:
   - JWT Token必须妥善保管，不要泄露
   - Token有过期时间，过期后需要重新登录
   - 不要在代码中硬编码Token

2. **验证码安全**:
   - 验证码接口仅限特定IP访问
   - 验证码有效期16秒，过期后需要重新获取
   - 不要在代码中硬编码验证码

3. **特殊密码安全**:
   - 特殊密码必须妥善保管，不要泄露
   - 不要在代码中硬编码特殊密码
   - 建议使用环境变量或配置中心管理

4. **参数验证**:
   - 所有必填参数必须提供
   - 参数格式必须符合要求
   - 参数范围必须在有效范围内

5. **数据一致性**:
   - 签到操作会立即生效
   - 签到记录永久保存到数据库
   - 使用Redis缓存防止重复签到

6. **性能优化**:
   - 排名查询使用数据库聚合，性能较好
   - 对于高频访问，可考虑使用Redis缓存
   - 建议合理设置查询数量，避免查询过多数据

7. **错误处理**:
   - 所有错误都会返回统一的错误格式
   - 错误信息会明确说明失败原因
   - 建议根据错误码进行相应的错误处理

8. **IP白名单**:
   - 验证码接口需要IP白名单认证
   - IP白名单配置在服务器端，不在本文档中暴露
   - 请勿公开IP白名单配置信息
