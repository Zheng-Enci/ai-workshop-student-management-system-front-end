# 特殊密码接口文档

## 文档概述

本文档描述了AI工作坊学生管理系统中所有需要特殊密码认证的API接口。这些接口提供了系统管理、学生管理、权限分配、考勤管理和积分管理等核心功能。

### 适用范围

- **目标用户**: 系统管理员（需要特殊密码权限）
- **权限级别**: 需要特殊密码认证
- **使用场景**: 系统管理、学生信息管理、权限分配、考勤补卡、积分调整

### 安全说明

**重要**: 本文档中的所有特殊密码、密钥等敏感信息均使用占位符表示。实际使用时请：

1. 联系系统管理员获取真实的特殊密码
2. 不要在代码中硬编码敏感信息
3. 使用环境变量或安全的配置管理方式
4. 定期更换特殊密码
5. 限制特殊密码的访问权限

## 基础信息

- **基础URL**: `http://<YOUR_SERVER_IP>:7001`
- **API版本**: v1
- **认证方式**: 特殊密码（通过Query参数或Header传递）
- **数据格式**: JSON
- **字符编码**: UTF-8

## 通用响应格式

所有接口都使用统一的响应格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200表示成功，其他表示失败 |
| message | String | 响应消息 |
| data | Object | 响应数据，具体结构根据接口而定 |

### HTTP状态码说明

| 状态码 | 说明 | 常见场景 |
|--------|------|----------|
| 200 | 操作成功 | 请求正常处理 |
| 400 | 请求参数错误或业务逻辑错误 | 特殊密码错误、参数验证失败、业务规则违反 |
| 401 | 认证失败 | Token无效（部分接口需要） |
| 500 | 服务器内部错误 | 数据库连接失败、系统异常 |

## 认证方式

### 特殊密码认证

需要特殊密码认证的接口使用特殊密码进行身份验证，有两种传递方式：

1. **Query参数方式**（部分接口）:
   ```
   ?special-password=<YOUR_SPECIAL_PASSWORD>
   ```

2. **Header方式**（RESTful接口）:
   ```
   X-Special-Password: <YOUR_SPECIAL_PASSWORD>
   ```

3. **请求体方式**（部分接口）:
   ```json
   {
     "specialPassword": "<YOUR_SPECIAL_PASSWORD>",
     ...
   }
   ```

**注意**: 特殊密码属于敏感信息，请妥善保管，不要泄露。

## 1. 学生管理接口

### 1.1 获取所有学生信息

**接口地址**: `GET /api/v1/students/all-with-special-password`

**接口描述**: 通过特殊密码获取系统中所有学生的完整信息，包括学生表主键ID。

**请求参数**:

| 参数名 | 类型 | 位置 | 必填 | 说明 |
|--------|------|------|------|------|
| special-password | String | Query | 是 | 特殊密码（请联系系统管理员获取） |

**请求示例**:

```bash
GET /api/v1/students/all-with-special-password?special-password=<YOUR_SPECIAL_PASSWORD>
```

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "获取所有学生信息成功",
  "data": [
    {
      "id": 123,
      "studentId": 2021001001,
      "name": "张三",
      "gender": "男",
      "phoneNumber": "13812345678",
      "college": "计算机学院",
      "grade": 3,
      "major": "软件工程",
      "classNum": 1
    },
    {
      "id": 124,
      "studentId": 2021001002,
      "name": "李四",
      "gender": "女",
      "phoneNumber": "13812345679",
      "college": "计算机学院",
      "grade": 2,
      "major": "计算机科学与技术",
      "classNum": 2
    }
  ]
}
```

失败响应 (400):
```json
{
  "code": 400,
  "message": "特殊密码错误",
  "data": null
}
```

**业务规则**:
- 需要提供正确的特殊密码
- 返回系统中所有学生的详细信息
- 包含学生表主键ID（`id`字段），便于后续操作
- 不包含密码等敏感信息
- 用于管理员查看所有学生信息

**安全特性**:
- 特殊密码验证：只有知道特殊密码的用户才能访问
- 数据脱敏：不返回密码等敏感信息
- 无需Token验证：直接通过特殊密码验证身份

### 1.2 设置学生等级

**接口地址**: `POST /api/v1/students/set-level`

**接口描述**: 通过特殊密码设置指定学生的权限等级。

**请求头**:
```
Content-Type: application/json
Accept: application/json
```

**请求参数**:

| 参数名 | 类型 | 位置 | 必填 | 说明 |
|--------|------|------|------|------|
| special-password | String | Query | 是 | 特殊密码 |

**请求体**:

```json
{
  "studentId": "2021001001",
  "levelCode": 2
}
```

**参数验证规则**:

| 字段 | 验证规则 | 错误信息 |
|------|----------|----------|
| studentId | 以20-30开头的10位数字 | 学号格式不正确 |
| levelCode | 0-3之间的整数 | 等级代码必须在 0-3 之间 |

**等级代码说明**:

| 等级代码 | 等级名称 | 说明 | 权限范围 |
|----------|----------|------|----------|
| 0 | 社团成员 | 默认等级 | 基础签到权限 |
| 1 | 普通成员 | 基础成员权限 | 签到 + 查看个人信息 |
| 2 | 核心成员 | 核心成员权限 | 普通成员权限 + 查看统计数据 |
| 3 | 管理员 | 管理员权限 | 所有权限 + 管理其他学生 |

**请求示例**:

```bash
POST /api/v1/students/set-level?special-password=<YOUR_SPECIAL_PASSWORD>
Content-Type: application/json

{
  "studentId": "2021001001",
  "levelCode": 2
}
```

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "设置成功",
  "data": {
    "message": "设置成功"
  }
}
```

失败响应 (400):
```json
{
  "code": 400,
  "message": "特殊密码错误",
  "data": null
}
```

或

```json
{
  "code": 400,
  "message": "该学号不存在",
  "data": null
}
```

**业务规则**:
- 需要提供正确的特殊密码进行身份验证
- 学号必须存在于系统中
- 等级代码必须在有效范围内（0-3）
- 设置成功后立即生效
- 等级信息存储在独立的student_level表中

**安全特性**:
- 特殊密码验证：只有知道特殊密码的管理员才能设置学生等级
- 学号验证：确保只能为存在的学生设置等级
- 等级范围验证：防止设置无效的等级代码

### 1.3 管理员更新学生信息

**接口地址**: `PUT /api/v1/students/update-with-special-password`

**接口描述**: 通过特殊密码更新指定学生的所有信息，包括学号、姓名、联系方式等。

**请求头**:
```
Content-Type: application/json
Accept: application/json
```

**请求参数**:

| 参数名 | 类型 | 位置 | 必填 | 说明 |
|--------|------|------|------|------|
| special-password | String | Query | 是 | 特殊密码 |
| student-id | String | Query | 是 | 要修改的学生学号（旧学号） |

**请求体**:

```json
{
  "name": "张三",
  "studentId": "2021001002",
  "gender": "男",
  "phoneNumber": "13812345678",
  "college": "计算机学院",
  "major": "软件工程",
  "grade": 3,
  "classNum": 1,
  "password": "newpassword123"
}
```

**参数验证规则**:

| 字段 | 验证规则 | 错误信息 |
|------|----------|----------|
| name | 不能为空 | 名字不能为空 |
| studentId | 以20-30开头的10位数字 | 学号格式不正确 |
| gender | 男或女 | 性别必须为男或女 |
| phoneNumber | 11位手机号格式（1[3456789]开头） | 手机号格式不正确 |
| college | 不能为空 | 学院不能为空 |
| major | 不能为空 | 专业不能为空 |
| grade | 1-5之间的整数 | 年级必须大于0且小于6 |
| classNum | 1-100之间的整数 | 班级必须大于0且小于101 |
| password | 6-16位字符 | 密码长度必须在6到16之间 |

**请求示例**:

```bash
PUT /api/v1/students/update-with-special-password?special-password=<YOUR_SPECIAL_PASSWORD>&student-id=2021001001
Content-Type: application/json

{
  "name": "张三",
  "studentId": "2021001002",
  "gender": "男",
  "phoneNumber": "13812345678",
  "college": "计算机学院",
  "major": "软件工程",
  "grade": 3,
  "classNum": 1,
  "password": "newpassword123"
}
```

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "修改成功",
  "data": true
}
```

失败响应 (400):
```json
{
  "code": 400,
  "message": "特殊密码错误",
  "data": null
}
```

**业务规则**:
- 需要提供正确的特殊密码进行身份验证
- 学号必须存在于系统中
- 可以修改学生的所有信息，包括学号
- 如果修改学号，新学号必须唯一
- 如果修改手机号，新手机号必须唯一
- 密码会被加密存储

**安全特性**:
- 特殊密码验证：只有知道特殊密码的管理员才能修改学生信息
- 学号验证：确保只能修改存在的学生信息
- 唯一性验证：确保学号和手机号的唯一性
- 参数验证：所有字段都进行严格的格式验证

## 3. 积分管理接口

### 3.1 创建积分记录

**接口地址**: `POST /api/v1/points`

**接口描述**: 通过特殊密码为指定学生创建积分记录，记录积分变更原因和分数。

**请求头**:
```
Content-Type: application/json
Accept: application/json
X-Special-Password: <YOUR_SPECIAL_PASSWORD>
```

**请求体**:

```json
{
  "adjustReason": "参加活动表现优秀",
  "changePoints": 10,
  "targetStudentInfoId": 123
}
```

**参数验证规则**:

| 字段 | 验证规则 | 错误信息 |
|------|----------|----------|
| adjustReason | 不能为空，最长500字符 | 改分理由不能为空 / 改分理由不能超过500个字符 |
| changePoints | 不能为空，整数 | 改分分数不能为空 |
| targetStudentInfoId | 不能为空，Long类型 | 目标学生数据库表主键 ID 不能为空 |

**请求示例**:

```bash
POST /api/v1/points
Content-Type: application/json
X-Special-Password: <YOUR_SPECIAL_PASSWORD>

{
  "adjustReason": "参加活动表现优秀",
  "changePoints": 10,
  "targetStudentInfoId": 123
}
```

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "积分记录创建成功",
  "data": null
}
```

失败响应 (400):
```json
{
  "code": 400,
  "message": "特殊密码错误",
  "data": null
}
```

或

```json
{
  "code": 400,
  "message": "改分理由不能为空",
  "data": null
}
```

**业务规则**:
- 需要提供正确的特殊密码（通过Header传递）
- 目标学生ID必须存在于系统中
- 积分分数支持正负值（加分或扣分）
- 改分理由必须详细说明原因
- 积分记录会永久保存到数据库
- 创建时间自动记录

**安全特性**:
- 特殊密码验证：只有知道特殊密码的管理员才能创建积分记录
- 参数验证：所有字段都进行严格的格式验证
- RESTful设计：使用Header传递认证信息，符合RESTful API规范
- 数据完整性：积分记录包含完整的变更原因和时间戳

**注意事项**:
- `targetStudentInfoId` 是学生表的主键ID，不是学号
- 可以通过 `GET /api/v1/students/all-with-special-password` 接口获取学生的ID
- 积分分数可以是正数（加分）或负数（扣分）
- 改分理由建议详细说明，便于后续追溯

## 最佳实践

### 1. 安全建议

- **保护特殊密码**: 不要在代码中硬编码特殊密码，使用环境变量或配置中心管理
- **定期更换密码**: 建议定期更换特殊密码，提高系统安全性
- **访问控制**: 限制知道特殊密码的人员数量，建立访问审计机制
- **HTTPS传输**: 生产环境必须使用HTTPS协议传输，防止密码泄露

### 2. 使用建议

- **批量操作**: 对于大量学生信息操作，建议分批处理，避免单次请求数据过大
- **错误处理**: 实现完善的错误处理机制，对接口返回的错误信息进行友好提示
- **日志记录**: 记录所有管理员操作日志，便于审计和问题追踪
- **数据备份**: 在进行重要操作前，建议先备份相关数据

### 3. 性能优化

- **缓存策略**: 对于频繁查询的学生信息，可以考虑使用缓存
- **分页查询**: 获取所有学生信息时，如果数据量大，建议实现分页功能
- **异步处理**: 对于耗时较长的操作，可以考虑异步处理
