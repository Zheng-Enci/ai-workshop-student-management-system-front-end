# AI Workshop Student Management System - FrontEnd 项目规则

## 一、代码规范

### 1.1 缩进与格式

- 使用 Tab 缩进，缩进级别为 4 个空格
- 最大行宽 120 个字符
- 行尾符使用 LF (Unix)
- 文件末尾必须换行
- 字符串使用单引号
- 语句结尾不使用分号
- 对象大括号内两侧有空格

### 1.2 文件命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| Vue 组件 | 帕斯卡命名法 | `DashboardPage.vue` |
| CSS/SCSS | 短横线命名法 | `dashboard-page.css` |
| JavaScript | 驼峰命名法 | `dashboardDataLoader.js` |
| API 文件 | 功能+Api 命名 | `AttendanceApi.js` |
| 配置文件 | Config 结尾 | `Config.js` |
| 工具类 | Utils 结尾 | `Utils.js` |
| 类型文件 | TypeScript 类型文件 | `types/index.ts` |

### 1.3 目录结构规范

```
src/
├── api/                    # API 接口定义
│   ├── ts/                # TypeScript API 接口
│   └── *.js               # JavaScript API 接口
├── assets/                 # 静态资源
│   └── styles/            # 全局样式
├── components/             # 公共组件
├── composables/            # Vue 组合式 API
│   └── admin/             # 管理员相关组合式函数
├── config/                 # 配置文件
├── router/                # 路由配置
├── stores/                # Pinia 状态管理
└── views/                  # 页面组件
    └── [PageName]/
        ├── *.vue          # 主/桌面端/移动端组件
        ├── css/           # 样式文件
        │   ├── desktop/   # 桌面端样式
        │   └── mobile/   # 移动端样式
        ├── js/            # 逻辑脚本
        └── ts/            # TypeScript 脚本
```

### 1.4 CSS 命名规范

- 使用 BEM 命名法或短横线命名法
- 移动端和桌面端样式分开文件管理
- 桌面端样式文件添加 `-desktop` 后缀
- 移动端样式文件添加 `-mobile` 后缀
- 样式类名添加页面前缀避免冲突

### 1.5 组件代码结构

```vue
<script setup>
/** *******************************************************************
 * 组件名称 - 核心逻辑层
 * 功能概述：
 * 1. 功能点一描述
 * 2. 功能点二描述
 ********************************************************************/

// ======================== 依赖导入区 ========================
// 第三方库导入

// ======================== 组件导入区 ========================
// 子组件导入

// ======================== 常量定义区 ========================
// 常量定义

// ======================== 响应式状态 ========================
// ref/reactive 定义

// ======================== 计算属性 ========================
// computed 定义

// ======================== 方法定义 ========================
// 函数定义

// ======================== 生命周期钩子 ========================
// onMounted/onUnmounted 等
</script>

<template>
    <!-- 模板内容 -->
</template>

<style scoped>
/* 样式内容 */
</style>
```

## 二、命名约定

### 2.1 变量命名

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 组件 ref | 驼峰命名 | `const gradeChart = ref(null)` |
| 常量 | 全大写下划线 | `const API_BASE_URL = '...'` |
| 布尔变量 | is/has/can 前缀 | `isLoading`, `hasError` |
| 事件处理 | on/handle 前缀 | `onClick`, `handleSubmit` |
| API 函数 | 驼峰命名 | `getGradeStatistics()` |
| 私有变量 | 下划线前缀 | `_privateMethod()` |

### 2.2 页面文件命名

| 用途 | 命名规则 | 示例 |
|------|----------|------|
| 页面主文件 | 页面名 + Page | `DashboardPage.vue` |
| 桌面端页面 | 页面名 + PageDesktop | `DashboardPageDesktop.vue` |
| 移动端页面 | 页面名 + PageMobile | `DashboardPageMobile.vue` |
| 页面样式 | 页面名 + Page | `DashboardPage.css` |
| 页面逻辑 | 页面名 + Page | `DashboardPage.js` |
| 页面配置 | 页面名 + PageConfig | `DashboardPageConfig.js` |
| 页面工具 | 页面名 + PageUtils | `DashboardPageUtils.js` |
| 页面数据加载器 | 页面名 + DataLoader | `DashboardDataLoader.ts` |

### 2.3 API 文件命名

- TypeScript API: `AttendanceApi.ts`, `PointsApi.ts`
- JavaScript API: `AttendanceApi.js`, `PointsApi.js`
- 兼容层 API: `attendance.js`, `points.js`
- 管理员 API: `StudentManagementApi.js`

## 三、Git 提交规范

### 3.1 提交信息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 3.2 Type 类型定义

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug 修复 |
| docs | 文档变更 |
| style | 代码格式调整 |
| refactor | 代码重构 |
| perf | 性能优化 |
| test | 测试相关 |
| build | 构建相关 |
| ci | CI 配置变更 |
| chore | 其他变更 |
| revert | 回滚 |
| types | 类型定义 |
| wip | 开发中 |
| release | 发布版本 |

### 3.3 提交规则

- type 必须为小写
- subject 不能为空
- subject 不能以句号结尾
- subject 最大 72 个字符
- header 最大 100 个字符

## 四、开发注意事项

- 所有代码必须添加高质量注释，注释越多越好
- 修改文件后必须更新日志
- 日志文件保存在 `Note/更新日志/`
- 日志命名格式：`YYYY-MM-DD-功能描述.md`
- 自动执行 `git add` 和 `git commit`，不执行 `git push`

## 五、技术选型

### 5.1 核心技术栈

- **框架**: Vue 3 + Composition API
- **路由**: Vue Router 4
- **状态管理**: Pinia
- **UI 组件库**: Element Plus
- **构建工具**: Vite
- **数据可视化**: ECharts
- **HTTP 请求**: Axios

### 5.2 开发工具

- **代码检查**: ESLint
- **代码格式化**: Prettier
- **CSS 检查**: Stylelint
- **Git Hooks**: Husky
- **暂存检查**: Lint-staged
- **提交规范**: Commitlint

## 六、更新日志规范

### 6.1 日志文件位置

- 根目录: `Note/更新日志/`
- 年度目录: `Note/更新日志/YYYY/`
- 月度目录: `Note/更新日志/YYYY/MM/`

### 6.2 日志文件命名

- 格式: `YYYY-MM-DD-功能描述.md`

### 6.3 日志内容格式

```markdown
# YYYY-MM-DD

## [文件或模块名]
- 变更描述

## [另一个文件或模块名]
- 变更描述
```
