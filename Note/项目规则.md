# AI Workshop Student Management System - FrontEnd Project Rules

## 一、代码规范

### 1.1 缩进与格式

| 项目 | 规范 |
|------|------|
| 缩进方式 | 使用 Tab 缩进 |
| 缩进级别 | 4 个空格 |
| 最大行宽 | 120 个字符 |
| 行尾符 | LF (Unix) |
| 文件末尾 | 必须换行 |
| 字符串引号 | 单引号优先 |
| 语句结尾 | 不使用分号 |
| 对象属性空格 | 大括号内两侧有空格 |

### 1.2 文件命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| Vue 组件 | 帕斯卡命名法 | `DashboardPage.vue` |
| CSS/SCSS | 短横线命名法 | `dashboard-page.css` |
| JavaScript | 驼峰命名法 | `dashboardDataLoader.js` |
| API 文件 | 功能+Api 命名 | `AttendanceApi.js` |
| 配置文件 | Config 结尾 | `Config.js` |
| 工具类 | Utils 结尾 | `Utils.js` |
| 类型文件 | TypeScript 类型文件 | `types/index.ts` |

### 1.3 目录结构规范

```
src/
├── api/                    # API 接口定义
│   ├── ts/                # TypeScript API 接口
│   │   ├── AttendanceApi.ts
│   │   ├── PointsApi.ts
│   │   └── StudentApi.ts
│   └── *.js               # JavaScript API 接口
├── assets/                 # 静态资源
│   ├── styles/            # 全局样式
│   └── *.png/gif          # 图片资源
├── components/             # 公共组件
├── composables/            # Vue 组合式 API
│   └── admin/             # 管理员相关组合式函数
├── config/                 # 配置文件
├── router/                # 路由配置
├── stores/                # Pinia 状态管理
│   ├── admin.js
│   ├── theme.js
│   └── user.js
└── views/                  # 页面组件
    └── [PageName]/
        ├── *.vue          # 主组件文件
        ├── *.vue          # 桌面端组件
        ├── *.vue          # 移动端组件
        ├── css/           # 样式文件
        │   ├── *.css      # 主样式
        │   ├── desktop/   # 桌面端样式
        │   └── mobile/   # 移动端样式
        ├── js/            # 逻辑脚本
        └── ts/            # TypeScript 脚本
```

### 1.4 CSS 命名规范

- 使用 BEM 命名法或短横线命名法
- 移动端和桌面端样式分开文件管理
- 桌面端样式文件添加 `-desktop` 后缀
- 移动端样式文件添加 `-mobile` 后缀
- 样式类名添加页面前缀避免冲突

## 三、Git 提交规范

### 3.1 提交信息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 3.2 Type 类型定义

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug 修复 |
| docs | 文档变更 |
| style | 代码格式调整 |
| refactor | 代码重构 |
| perf | 性能优化 |
| test | 测试相关 |
| build | 构建相关 |
| ci | CI 配置变更 |
| chore | 其他变更 |
| revert | 回滚 |
| types | 类型定义 |
| wip | 开发中 |
| release | 发布版本 |

### 3.3 提交规则

- type 必须为小写
- subject 不能为空
- subject 不能以句号结尾
- subject 最大 72 个字符
- header 最大 100 个字符
- body 每行最大 100 个字符
- body 前必须有空行

### 3.4 Git Hooks (Husky + Lint-staged)

- 使用 Husky 管理 Git hooks
- 使用 Lint-staged 对暂存文件进行质量检查
- JavaScript/Vue 文件：ESLint 修复 + Prettier 格式化
- CSS/SCSS 文件：Stylelint 修复 + Prettier 格式化
- JSON/MD 文件：Prettier 格式化

## 四、开发流程规范

### 4.1 npm 脚本命令

| 命令 | 说明 |
|------|------|
| npm run dev | 启动开发服务器 |
| npm run build | 生产环境构建 |
| npm run build:analyze | 构建并分析打包结果 |
| npm run preview | 预览生产构建 |
| npm run serve:prod | 生产环境服务 |
| npm run lint | ESLint 检查 |
| npm run lint:fix | ESLint 修复 |
| npm run lint:css | Stylelint 检查 |
| npm run lint:css:fix | Stylelint 修复 |
| npm run format | Prettier 格式化 |
| npm run format:check | Prettier 检查格式 |
| npm run check | 代码全面检查 |
| npm run check:deps | 依赖检查 |
| npm run check:security | 安全审计 |
| npm run check:all | 全部检查 |
| npm run check:quality | 代码质量检查 |

### 4.2 代码质量检查流程

1. 开发时运行 `npm run dev` 自动触发代码质量检查
2. 提交前运行 `npm run check` 全面检查
3. 提交时 Lint-staged 自动处理暂存文件
4. Commitlint 检查提交信息格式

### 4.3 开发注意事项

- 所有代码必须添加高质量注释
- 注释越多越好，描述要准确
- 修改文件后必须更新日志
- 日志文件保存在 `Note/更新日志/`
- 日志命名格式：`YYYY-MM-DD-功能描述.md`
- 自动执行 `git add` 和 `git commit`，不执行 `git push`

## 五、命名约定

### 5.1 变量命名

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 组件 ref | 驼峰命名 | `const gradeChart = ref(null)` |
| 常量 | 全大写下划线 | `const API_BASE_URL = '...'` |
| 布尔变量 | is/has/can 前缀 | `isLoading`, `hasError` |
| 事件处理 | on/handle 前缀 | `onClick`, `handleSubmit` |
| API 函数 | 驼峰命名 | `getGradeStatistics()` |
| 私有变量 | 下划线前缀 | `_privateMethod()` |

### 5.2 文件命名对照表

| 用途 | 命名规则 | 示例 |
|------|----------|------|
| 页面主文件 | 页面名 + Page | `DashboardPage.vue` |
| 桌面端页面 | 页面名 + PageDesktop | `DashboardPageDesktop.vue` |
| 移动端页面 | 页面名 + PageMobile | `DashboardPageMobile.vue` |
| 页面样式 | 页面名 + Page | `DashboardPage.css` |
| 桌面端样式 | 页面名 + PageDesktop | `DashboardPageDesktop.css` |
| 移动端样式 | 页面名 + PageMobile | `DashboardPageMobile.css` |
| 页面逻辑 | 页面名 + Page | `DashboardPage.js` |
| 页面配置 | 页面名 + PageConfig | `DashboardPageConfig.js` |
| 页面工具 | 页面名 + PageUtils | `DashboardPageUtils.js` |
| 页面数据加载器 | 页面名 + DataLoader | `DashboardDataLoader.ts` |

### 5.3 API 文件命名

- TypeScript API: `AttendanceApi.ts`, `PointsApi.ts`
- JavaScript API: `AttendanceApi.js`, `PointsApi.js`
- 兼容层 API: `attendance.js`, `points.js`
- 管理员 API: `StudentManagementApi.js`

## 八、其他规范

### 8.1 图片资源命名

- 使用有意义的英文命名
- 使用下划线分隔单词
- 示例: `AiWorkShop_icon.png`, `ErWeiMa.png`

### 8.2 组件引入规范

- Element Plus 图标按需引入
- Element Plus 组件按需引入

- 使用 Pinia 进行状态管理
- 管理员状态: `stores/admin.js`

- 使用 Vue Router 4
- 路由配置: `src/router/index.js`
- 支持懒加载路由

### 8.5 环境配置

- API 地址配置: `src/config/index.js`
- 不同环境使用不同配置
