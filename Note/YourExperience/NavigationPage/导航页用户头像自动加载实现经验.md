# 导航页用户头像自动加载实现经验

## 概述

本文档总结了在导航页面（电脑端和手机端）实现用户头像自动加载功能的经验，包括接口封装、头像加载逻辑、错误处理、UI设计、企业级参考案例等方面的最佳实践。

## 需求分析

### 功能需求

在导航页面的用户信息区域自动加载并显示用户头像：
- **自动加载**：页面加载时自动获取并显示用户头像
- **无头像提示**：如果用户没有上传头像，显示友好的提示信息引导用户上传
- **错误处理**：头像加载失败时显示默认图标
- **多端适配**：支持手机端和电脑端独立实现

### 数据来源

1. **学生数据库ID**：`GET /api/v1/students/get-student-database-table-id`（需要Token）
2. **头像URL**：`GET /api/v1/students/avatar/{id}`（根据学生数据库表主键ID获取头像图片）

## 技术实现

### 1. 接口封装

#### 1.1 上传头像接口

在`src/api/student.js`中新增：

```javascript
/**
 * 上传学生头像
 * @param {String} token - JWT token
 * @param {File} file - 头像文件
 * @returns {Promise} Promise对象
 */
export const uploadAvatar = async (token, file) => {
  try {
    const formData = new FormData()
    formData.append('file', file)
    
    const response = await api.post('/api/v1/students/avatar', formData, {
      params: { token },
      headers: {
        'Content-Type': 'multipart/form-data'
      }
    })
    return response.data
  } catch (error) {
    // 完善的错误处理
  }
}
```

**接口说明**：
- 接口地址：`POST /api/v1/students/avatar`
- 参数：`token`（Query参数，JWT token）、`file`（FormData，头像文件）
- 返回：`data` 为字符串，表示上传成功消息
- 需要JWT Token认证

#### 1.2 获取头像URL辅助函数

```javascript
/**
 * 获取头像URL（辅助函数）
 * @param {Number} studentInfoId - 学生数据库表主键ID
 * @returns {String} 头像URL
 */
export const getAvatarUrl = (studentInfoId) => {
  if (!studentInfoId) return null
  return `${config.API_BASE_URL}/api/v1/students/avatar/${studentInfoId}`
}
```

**关键点**：
- 使用API基础URL构造完整头像URL
- 如果ID为空返回null，避免无效请求
- 头像URL可以直接作为img标签的src使用

### 2. 头像加载逻辑

#### 2.1 加载流程

使用Image对象预加载检测头像是否存在：

```javascript
const loadUserAvatar = async () => {
  try {
    avatarLoading.value = true
    const token = localStorage.getItem('token')
    if (!token) {
      showDefaultAvatar()
      avatarLoading.value = false
      return
    }

    // 获取学生数据库ID
    const idResponse = await getStudentDatabaseTableId(token)
    if (idResponse.code !== 200 || !idResponse.data) {
      showDefaultAvatar()
      avatarLoading.value = false
      return
    }

    const studentInfoId = idResponse.data
    const avatarUrlString = getAvatarUrl(studentInfoId)
    
    if (!avatarUrlString) {
      showDefaultAvatar()
      avatarLoading.value = false
      return
    }

    // 预加载图片检测
    const img = new Image()
    img.onload = () => {
      avatarUrl.value = avatarUrlString
      hasAvatar.value = true
      showUploadTip.value = false
      avatarLoading.value = false
    }
    img.onerror = () => {
      showDefaultAvatar()
      avatarLoading.value = false
    }
    img.src = avatarUrlString
  } catch (error) {
    if (error.message.includes('Token无效') || error.message.includes('请重新登录')) {
      localStorage.removeItem('token')
      localStorage.removeItem('userInfo')
      router.push('/login')
    } else {
      showDefaultAvatar()
    }
    avatarLoading.value = false
  }
}
```

**关键点**：
1. **预加载检测**：使用`Image`对象预加载头像，通过`onload`和`onerror`事件判断头像是否存在
2. **状态管理**：使用`avatarLoading`控制加载状态，`hasAvatar`控制是否显示头像，`showUploadTip`控制是否显示上传提示
3. **错误处理**：Token失效时跳转登录页，其他错误显示默认头像
4. **性能优化**：只在有Token时才加载，避免无效请求

#### 2.2 默认头像显示

```javascript
const showDefaultAvatar = () => {
  hasAvatar.value = false
  showUploadTip.value = true
  avatarUrl.value = null
}
```

**关键点**：
- 设置`hasAvatar`为false，显示默认图标
- 设置`showUploadTip`为true，显示上传提示
- 清空`avatarUrl`，避免显示错误图片

### 4. 错误处理策略

参考大厂实现，采用以下策略：

#### 4.1 图片加载检测

使用`Image`对象预加载，通过事件判断：

```javascript
const img = new Image()
img.onload = () => {
  // 头像存在，显示头像
  avatarUrl.value = avatarUrlString
  hasAvatar.value = true
}
img.onerror = () => {
  // 头像不存在或加载失败，显示默认图标
  showDefaultAvatar()
}
img.src = avatarUrlString
```

**优势**：
- 不依赖HTTP状态码，直接检测图片是否能加载
- 避免404错误在控制台显示
- 用户体验更流畅

#### 4.2 接口错误处理

- **400错误**（头像不存在）：显示默认图标和上传提示，不显示错误信息
- **401错误**（Token无效）：跳转登录页，清除本地存储
- **网络错误**：显示默认图标，不显示错误提示（避免干扰用户）

#### 4.3 缓存机制

- 头像URL存储在响应式变量中
- 避免重复请求
- 页面刷新时重新加载
