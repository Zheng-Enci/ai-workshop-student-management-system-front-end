# 导航页积分展示实现经验

## 概述

本文档总结了在导航页面（电脑端和手机端）实现积分展示功能的经验，包括接口封装、数据计算、UI设计、企业级参考案例等方面的最佳实践。

## 需求分析

### 功能需求

在导航页面的用户信息区域展示学生的积分信息：
- **总积分**：签到积分 + 活动积分
- **活动积分**：从积分系统获取的活动积分总和
- **签到积分**：签到次数 × 0.64（四舍五入）

### 数据来源

1. **签到次数**：`GET /api/v1/attendance/my-attendance-count`（需要Token）
2. **学生数据库ID**：`GET /api/v1/students/get-student-database-table-id`（需要Token）
3. **活动积分**：`GET /api/v1/points/total-points-by-student-info-id`（需要学生数据库ID）

## 技术实现

### 1. 接口封装

#### 1.1 获取学生活动积分接口

在`src/api/points.js`中新增：

```javascript
export const getTotalPointsByStudentInfoId = async (studentInfoId) => {
  try {
    const response = await api.get('/api/v1/points/total-points-by-student-info-id', {
      params: {
        'target-student-info-id': studentInfoId
      }
    })
    return response.data
  } catch (error) {
    // 完善的错误处理
  }
}
```

**接口说明**：
- 接口地址：`GET /api/v1/points/total-points-by-student-info-id`
- 参数：`target-student-info-id`（学生数据库表主键ID，必填）
- 返回：`data` 为 Long 类型，表示活动积分总和
- 如果没有积分记录，返回0

### 2. 数据加载逻辑

#### 2.1 并行加载优化

使用`Promise.all`并行加载签到次数和学生数据库ID，减少等待时间：

```javascript
const loadPoints = async () => {
  try {
    pointsLoading.value = true
    const token = localStorage.getItem('token')
    if (!token) {
      pointsLoading.value = false
      return
    }

    // 并行加载签到次数和学生数据库ID
    const [attendanceResponse, studentIdResponse] = await Promise.all([
      getMyAttendanceCount(token).catch(() => ({ code: 0, data: { count: 0 } })),
      getStudentDatabaseTableId(token).catch(() => ({ code: 0, data: null }))
    ])

    // 计算签到积分
    if (attendanceResponse.code === 200) {
      signInPoints.value = Math.round(attendanceResponse.data.count * 0.64)
    } else {
      signInPoints.value = 0
    }

    // 获取活动积分
    if (studentIdResponse.code === 200 && studentIdResponse.data) {
      const activityResponse = await getTotalPointsByStudentInfoId(studentIdResponse.data).catch(() => ({ code: 0, data: 0 }))
      if (activityResponse.code === 200) {
        activityPoints.value = activityResponse.data || 0
      } else {
        activityPoints.value = 0
      }
    } else {
      activityPoints.value = 0
    }

    // 计算总积分
    totalPoints.value = (signInPoints.value || 0) + (activityPoints.value || 0)
  } catch (error) {
    // 错误处理
  } finally {
    pointsLoading.value = false
  }
}
```

**关键点**：
1. 使用`Promise.all`并行加载，提升性能
2. 使用`.catch()`处理单个接口失败，避免整体失败
3. 错误时设置默认值0，避免显示null
4. 使用`finally`确保loading状态正确更新

#### 2.2 积分计算规则

- **签到积分**：`Math.round(attendanceCount * 0.64)`（四舍五入）
- **活动积分**：直接从接口获取，如果没有记录返回0
- **总积分**：`signInPoints + activityPoints`

### 3. UI设计实现

#### 3.1 电脑端设计

**布局**：
- 使用多列布局（flex布局，gap: 16px）
- 卡片尺寸：min-width: 140px
- 字体大小：标签12px，数值20px

**样式特点**：
- 毛玻璃效果：`backdrop-filter: blur(20px)`
- 渐变背景：每个积分类型使用不同的渐变色彩
  - 总积分：紫色渐变（#667eea → #764ba2）
  - 活动积分：金色渐变（#ffc107 → #ff9800）
  - 签到积分：绿色渐变（#4caf50 → #388e3c）
- Hover效果：`transform: translateY(-4px)`，增强背景透明度

**代码示例**：

```vue
<div class="points-display">
  <div class="points-card total-points">
    <div class="points-icon">
      <el-icon size="20"><Coin /></el-icon>
    </div>
    <div class="points-content">
      <div class="points-label">总积分</div>
      <div class="points-value">{{ totalPoints !== null ? totalPoints : 0 }}</div>
    </div>
  </div>
  <!-- 其他积分卡片 -->
</div>
```

## 关键经验总结

### 1. 接口封装

- **错误处理**：完善的错误处理机制，区分不同错误类型
- **参数验证**：确保参数格式正确
- **返回值处理**：处理null和undefined情况

### 2. 数据加载

- **并行加载**：使用`Promise.all`并行加载多个接口，提升性能
- **容错处理**：单个接口失败不影响其他接口
- **默认值设置**：错误时设置合理的默认值，避免显示null

### 3. UI设计

- **多端适配**：电脑端和手机端完全独立实现，不共享资源
- **视觉层次**：使用卡片、图标、数值分层展示
- **品牌一致性**：与现有导航页风格保持一致
- **交互反馈**：加载状态、错误提示、数据更新动画

### 4. 性能优化

- **并行请求**：使用`Promise.all`减少等待时间
- **条件渲染**：只在数据加载完成后显示积分卡片
- **错误边界**：完善的错误处理，避免页面崩溃
