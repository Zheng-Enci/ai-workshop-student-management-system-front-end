# 首页共用逻辑模块实现经验

## 问题背景
在 Vue3 项目中，为了提高代码复用性和维护性，需要将多个组件共享的逻辑抽取到独立的 JS 文件中。首页作为系统的入口，包含了许多通用功能如主题切换、导航跳转、联系表单等。

## 解决方案
使用 Vue3 Composition API 创建独立的逻辑模块，通过 `useXXX` 命名规范导出组合式函数。

## 实现要点

### 1. 文件结构
```
src/
└── views/
    └── HomePage/
        ├── HomePage.js          # 共用逻辑模块
        ├── HomePage.vue         # 路由分发组件
        ├── HomePageDesktop.vue  # 桌面版首页
        └── HomePageMobile.vue   # 移动版首页
```

### 2. 核心实现模式
```javascript
import { ref, onMounted } from 'vue'
import { useRouter } from 'vue-router'
import { useThemeStore } from '@/stores/theme'

export const useHomePageLogic = () => {
  // 状态管理
  const router = useRouter()
  const themeStore = useThemeStore()
  const contactFormVisible = ref(false)
  
  // 方法实现
  const toggleTheme = () => {
    themeStore.toggleTheme()
  }
  
  const goToLogin = () => {
    router.push('/login')
  }
  
  // 生命周期钩子
  onMounted(() => {
    // 初始化逻辑
  })
  
  // 返回需要的属性和方法
  return {
    toggleTheme,
    contactFormVisible,
    goToLogin
  }
}
```

### 3. 使用方式
```vue
<script setup>
import { useHomePageLogic } from './HomePage.js'

const { 
  toggleTheme, 
  contactFormVisible, 
  goToLogin 
} = useHomePageLogic()
</script>
```

## 最佳实践

1. **命名规范**：使用 `useXXX` 命名组合式函数，符合 Vue3 生态约定
2. **功能聚合**：将相关联的功能组织在一起，便于维护和理解
3. **状态管理**：合理使用响应式 API 和 Pinia 状态管理
4. **生命周期**：在逻辑模块中处理组件生命周期相关逻辑
5. **错误处理**：对异步操作进行适当的错误处理

## 注意事项

1. 确保导入的依赖项在项目中存在且路径正确
2. 对于异步操作，要考虑加载状态和错误处理
3. 避免在逻辑模块中直接操作 DOM，保持逻辑的纯净性
4. 合理划分逻辑边界，避免将不相关的功能放在同一模块中
