# 积分调整功能实现经验

## 概述

本文档总结了在超级管理员页面实现积分调整功能的经验，包括接口封装、UI设计、表单验证、错误处理等方面的最佳实践。

## 功能需求

在超级管理员页面为每个学生添加"修改积分"功能，允许管理员为学生创建积分记录，记录积分变更原因和分数。

## 技术实现

### 1. 接口封装

#### 1.1 创建API文件

在 `src/api/points.js` 中创建积分调整接口封装：

```javascript
import axios from 'axios'
import config from '@/config'

const api = axios.create({
  baseURL: config.API_BASE_URL,
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json'
  }
})

export const createPointsRecord = async (specialPassword, adjustReason, changePoints, targetStudentInfoId) => {
  try {
    const response = await api.post('/api/v1/points', {
      adjustReason: adjustReason,
      changePoints: changePoints,
      targetStudentInfoId: targetStudentInfoId
    }, {
      headers: {
        'X-Special-Password': specialPassword
      }
    })
    return response.data
  } catch (error) {
    // 错误处理
  }
}
```

#### 1.2 关键要点

- **特殊密码传递**：使用 `X-Special-Password` Header传递特殊密码，符合RESTful API设计规范
- **参数顺序**：函数参数顺序与接口文档一致（specialPassword, adjustReason, changePoints, targetStudentInfoId）
- **错误处理**：实现完善的错误处理，区分网络错误、服务器错误、业务错误
- **超时设置**：设置10秒超时，避免长时间等待

### 5. 函数实现

#### 5.1 打开对话框

```javascript
const openPointsDialog = (student) => {
  if (!student || !student.id) {

```

**关键要点**：

- **表单重置**：打开对话框时重置表单数据
- **状态设置**：设置选中学生和对话框可见性

#### 5.2 取消操作

```javascript
const cancelPoints = () => {
  pointsDialogVisible.value = false
  pointsFormRef.value?.resetFields()
  pointsSelectedStudent.value = null
  pointsForm.value = {
    changePoints: null,
    adjustReason: ''
  }
}
```

**关键要点**：
- **表单重置**：使用 `resetFields()` 重置表单验证状态
- **状态清理**：清理所有相关状态变量
- **可选链操作**：使用 `?.` 避免表单引用不存在时的错误

#### 5.3 确认提交

```javascript
const confirmPoints = async () => {
  if (!pointsFormRef.value) return
  
  try {
    await pointsFormRef.value.validate()
    
    const adminPassword = adminStore.getAdminPassword()
    if (!adminPassword) {
      ElMessage.error('身份验证已过期，请重新登录')
      router.push('/admin/auth')
      return
    }

    if (!pointsSelectedStudent.value || !pointsSelectedStudent.value.id) {
      ElMessage.error('学生信息不完整')
      return
    }

    pointsLoading.value = true
    
    const response = await createPointsRecord(
      adminPassword,
      pointsForm.value.adjustReason.trim(),
      parseInt(pointsForm.value.changePoints),
      pointsSelectedStudent.value.id
    )
    
    if (response.code === 200) {
      ElMessage.success('积分记录创建成功')
      pointsDialogVisible.value = false
      pointsFormRef.value?.resetFields()
      pointsSelectedStudent.value = null
    } else {
      ElMessage.error(response.message || '积分记录创建失败')
    }
  } catch (error) {
    if (error.message) {
      ElMessage.error(error.message)
    }
  } finally {
    pointsLoading.value = false
  }
}
```

**关键要点**：
- **表单验证**：先验证表单，验证通过后再提交
- **身份验证**：检查特殊密码是否存在，过期时跳转到登录页
- **数据验证**：检查学生信息是否完整
- **防重复提交**：使用 `pointsLoading` 防止重复提交
- **数据转换**：
  - 改分理由使用 `trim()` 去除首尾空格
  - 积分变动值使用 `parseInt()` 转换为整数
  - 使用学生对象的 `id` 字段（数据库主键）而非 `studentId`（学号）
- **错误处理**：完善的错误处理，显示友好的错误提示
- **成功处理**：成功后关闭对话框、重置表单、清理状态

### 7. 错误处理

```javascript
catch (error) {
  if (error.response) {
    const status = error.response.status

    } else if (status >= 500) {
      throw new Error('服务器错误，请稍后重试')
```

### 9. 注意事项

#### 9.1 学生ID使用

- **数据库主键**：使用学生表的数据库主键ID（`id`字段），而非学号（`studentId`字段）
- **数据获取**：通过 `getAllStudentsWithSpecialPassword` 接口获取学生数据，该接口返回的数据包含 `id` 字段
- **数据验证**：在使用学生ID前，检查学生对象和ID是否存在

#### 9.2 特殊密码管理

- **内存存储**：特殊密码存储在Pinia store的内存中，不存储在localStorage或sessionStorage
- **过期检查**：每次调用接口前检查特殊密码是否存在，过期时跳转到登录页
- **安全传输**：使用Header传递特殊密码，符合RESTful API设计规范

#### 9.3 表单重置

- **打开时重置**：打开对话框时重置表单数据
- **取消时重置**：取消操作时重置表单验证状态
- **成功后重置**：操作成功后重置表单数据
