# 热力图数据计算问题修复经验

## 问题描述
超级管理员页面的热力图显示次数变少，用户反馈热力图数据不准确。

## 问题分析
经过深入分析发现，热力图数据变少的根本原因是：

1. **数据源错误**：热力图使用了`filteredStudentAttendanceRecords`（过滤后的数据）
2. **月份过滤限制**：`filteredStudentAttendanceRecords`只显示当前月份的数据
3. **数据范围不完整**：热力图应该显示所有历史数据，而不是仅当前月份

### 技术细节
- `filteredStudentAttendanceRecords`是一个computed属性，只返回当前月份的数据
- 热力图和折线图应该显示学生的所有考勤记录，不受月份限制
- 日历组件使用过滤后的数据是正确的，但图表应该使用完整数据

## 解决方案

### 1. 修改热力图数据生成函数
```javascript
const generateHeatmapData = () => {
  const data = []
  const weekDays = ['周一', '周二', '周三', '周四', '周五', '周六', '周日']
  const timeSlots = ['上午', '下午', '晚上']
  
  // 使用完整数据而不是过滤后的数据
  const records = allStudentAttendanceRecords.value || []
  
  if (!records || !Array.isArray(records) || records.length === 0) {
    weekDays.forEach((day, dayIndex) => {
      timeSlots.forEach((slot, slotIndex) => {
        data.push([dayIndex, slotIndex, 0])
      })
    })
    return data
  }
  
  weekDays.forEach((day, dayIndex) => {
    timeSlots.forEach((slot, slotIndex) => {
      let count = 0
      records.forEach(record => {
        if (!record || !record.attendanceDateTime) return
        
        const date = new Date(record.attendanceDateTime)
        if (isNaN(date.getTime())) return
        
        const dayOfWeek = date.getDay() === 0 ? 6 : date.getDay() - 1
        const hour = date.getHours()
        
        if (dayOfWeek === dayIndex) {
          if (slot === '上午' && hour >= 8 && hour < 11) count++
          else if (slot === '下午' && hour >= 14 && hour < 17) count++
          else if (slot === '晚上' && hour >= 19 && hour < 22) count++
        }
      })
      data.push([dayIndex, slotIndex, count])
    })
  })
  
  return data
}
```

### 2. 修改折线图数据生成函数
```javascript
const generateLineData = () => {
  const dateMap = new Map()
  
  // 使用完整数据而不是过滤后的数据
  const records = allStudentAttendanceRecords.value || []
  
  if (!records || !Array.isArray(records) || records.length === 0) {
    return {
      dates: [],
      values: []
    }
  }
  
  records.forEach(record => {
    if (!record || !record.attendanceDateTime) return
    
    const date = new Date(record.attendanceDateTime)
    if (isNaN(date.getTime())) return
    
    const dateStr = date.toISOString().split('T')[0]
    dateMap.set(dateStr, (dateMap.get(dateStr) || 0) + 1)
  })
  
  const sortedDates = Array.from(dateMap.keys()).sort()
  const dailyValues = sortedDates.map(date => dateMap.get(date))
  
  let cumulativeSum = 0
  const cumulativeValues = dailyValues.map(value => {
    cumulativeSum += value
    return cumulativeSum
  })
  
  return {
    dates: sortedDates,
    values: cumulativeValues
  }
}
```

## 技术要点

### 关键发现
1. **数据源选择错误**：图表应该使用完整数据，不是过滤后的数据
2. **computed属性限制**：`filteredStudentAttendanceRecords`只适用于日历显示
3. **数据范围理解**：热力图和折线图需要显示所有历史数据

### 解决策略
1. **直接使用原始数据**：`allStudentAttendanceRecords.value`包含所有考勤记录
2. **分离数据用途**：日历使用过滤数据，图表使用完整数据
3. **保持数据一致性**：确保图表显示的数据是准确的

### 最佳实践
1. **明确数据用途**：不同组件使用不同的数据源
2. **避免数据重复过滤**：图表不需要月份过滤
3. **保持数据完整性**：图表应该显示完整的历史数据

## 预期效果

- ✅ 折线图显示完整的签到趋势
- ✅ 数据统计准确，不再出现次数变少的问题
