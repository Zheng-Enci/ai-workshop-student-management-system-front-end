## 超级管理员页面主题切换功能修复

### 问题
超级管理员页面的主题切换模式有问题，具体表现为：
1. AI坊图标缺少点击事件，无法切换主题
2. 页面缺少夜间模式的CSS样式定义
3. 没有导入和使用主题store

### 分析
技术指令要求所有页面都必须有白天模式和夜间模式切换功能，通过点击 src/assets/AiWorkShop_icon.png 图标实现。超级管理员页面作为特殊的安全隐私保护页面，需要完整的主题支持。

```vue
<img 
  src="@/assets/AiWorkShop_icon.png" 
  alt="AI坊"

  @click="toggleTheme"
  title="切换主题模式"
>
```

#### 2. 导入主题store
```javascript
import { useThemeStore } from '@/stores/theme'

const themeStore = useThemeStore()
const { toggleTheme } = themeStore
```

## CSS变量主题系统重构

### 问题
用户反馈"很多字还是看不清"，经过分析发现超级管理员页面还在使用硬编码的颜色值，而其他页面已经采用了CSS变量的方式实现主题切换。

### 分析
通过学习其他页面的实现方式和联网学习最佳实践，发现：
1. **硬编码问题**：使用固定的颜色值，无法动态响应主题变化
2. **维护困难**：需要为每个元素单独定义夜间模式样式
3. **不一致性**：与其他页面的主题实现方式不统一
4. **可读性问题**：文字颜色对比度不够，特别是在白色背景上

### 解决方案

#### 1. 建立CSS变量系统
```css
:root {
  --admin-bg-primary: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
  --admin-bg-secondary: rgba(255, 255, 255, 0.95);
  --admin-bg-card: rgba(255, 255, 255, 0.95);
  --admin-text-primary: #2c3e50;
  --admin-text-secondary: #5a6c7d;
  --admin-text-tertiary: #7f8c8d;
  --admin-border-color: rgba(255, 255, 255, 0.2);
  --admin-shadow: rgba(0, 0, 0, 0.1);
}

html.dark {
  --admin-bg-primary: linear-gradient(135deg, #1a1a2e 0%, #16213e 100%);
  --admin-bg-secondary: rgba(0, 0, 0, 0.3);
  --admin-bg-card: rgba(0, 0, 0, 0.8);
  --admin-text-primary: #ffffff;
  --admin-text-secondary: #e2e8f0;
  --admin-text-tertiary: #cbd5e1;
  --admin-border-color: rgba(255, 255, 255, 0.1);
  --admin-shadow: rgba(0, 0, 0, 0.2);
}
```

#### 2. 应用CSS变量到组件
```css
.admin-container {
  background: var(--admin-bg-primary);
}

.auth-card {
  background: var(--admin-bg-card);
  border: 1px solid var(--admin-border-color);
  box-shadow: 0 20px 40px var(--admin-shadow);
}

.auth-header h2 {
  color: var(--admin-text-primary);
}

.auth-header p {
  color: var(--admin-text-secondary);
  font-weight: 500;
}

.students-section {
  background: var(--admin-bg-card);
  border: 1px solid var(--admin-border-color);
}

.section-header h2 {
  color: var(--admin-text-primary);
}

.student-name {
  color: var(--admin-text-primary);
}

.phone-number {
  color: var(--admin-text-secondary);
  font-weight: 500;
}

.college, .major {
  color: var(--admin-text-primary);
}

.class-num {
  color: var(--admin-text-secondary);
  font-weight: 500;
}
```

#### 3. 删除冗余的夜间模式样式
删除了所有 `html.dark` 选择器的样式定义，因为现在通过CSS变量自动处理主题切换。

### 效果
- ✅ **主题切换流畅**：CSS变量自动响应主题变化
- ✅ **文字清晰可读**：统一的文字颜色标准，确保对比度
- ✅ **代码简洁**：减少了大量重复的样式定义
- ✅ **维护性强**：只需修改CSS变量即可调整整个主题
- ✅ **一致性**：与其他页面的主题实现方式统一

### 关键要点
1. **CSS变量优势**：
   - 动态响应主题变化
   - 减少代码重复
   - 便于维护和修改
   - 性能更好

2. **颜色标准**：
   - 日间模式主要文字：`--admin-text-primary: #2c3e50`（深色）
   - 日间模式辅助文字：`--admin-text-secondary: #5a6c7d`（中等深度）
   - 夜间模式主要文字：`--admin-text-primary: #ffffff`（白色）
   - 夜间模式辅助文字：`--admin-text-secondary: #e2e8f0`（浅色）

3. **背景策略**：
   - 日间模式：紫蓝色渐变背景，白色半透明卡片
   - 夜间模式：深蓝色渐变背景，黑色半透明卡片
   - 卡片背景：根据主题模式动态调整，确保文字可读性

4. **对比度优化**：
   - 日间模式：白色背景配深色文字，确保高对比度
   - 夜间模式：深色背景配白色文字，确保高对比度
   - 添加字体粗细增强可读性
   - 符合WCAG可访问性标准

## CSS变量主题系统最佳实践

### 1. 变量命名规范
```css
/* 背景相关 */
--admin-bg-primary: 主背景
--admin-bg-secondary: 次要背景
--admin-bg-card: 卡片背景

/* 文字相关 */
--admin-text-primary: 主要文字
--admin-text-secondary: 辅助文字
--admin-text-tertiary: 次要文字

/* 边框和阴影 */
--admin-border-color: 边框颜色
--admin-shadow: 阴影颜色
```

### 2. 主题切换策略
- **日间模式**：使用 `:root` 定义默认变量
- **夜间模式**：使用 `html.dark` 覆盖变量值
- **平滑过渡**：添加 `transition` 属性实现平滑切换

### 3. 可访问性考虑
- **对比度**：确保文字与背景的对比度至少4.5:1
- **颜色独立性**：不依赖颜色传达重要信息
- **字体大小**：确保文字大小适合阅读

### 4. 性能优化
- **CSS变量**：比类名切换性能更好
- **减少重绘**：只改变CSS变量值，不重新计算样式
- **缓存友好**：CSS变量可以被浏览器缓存

const isDataLoaded = ref(false)
const loadingProgress = ref(0)

const loadAllData = async (specialPassword) => {
  const totalSteps = 5
  let currentStep = 0
