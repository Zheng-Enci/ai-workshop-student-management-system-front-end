# AdminPage - 日期选择器月份名称中文化修复经验

## 场景
超级管理员页面补卡功能中，Element Plus 日期选择器虽然配置了 `:locale="zhCn"`，但月份名称仍然显示为英文（如 "December"），需要动态替换为中文（如 "十二月"）。

## 问题原因
Element Plus 的日期选择器在某些情况下，即使配置了中文 locale，月份名称仍可能显示为英文。这可能是因为：
1. 日期选择器面板是动态渲染的，locale 配置可能未完全生效
2. 月份名称的显示可能依赖于浏览器的本地化设置
3. 日期选择器面板的头部标签（`.el-date-picker__header-label`）可能直接使用了英文月份名称

## 解决方案

### 1. 创建月份名称映射表
在组件中定义英文月份到中文月份的映射：

```javascript
const monthMap = {
  'January': '一月',
  'February': '二月',
  'March': '三月',
  'April': '四月',
  'May': '五月',
  'June': '六月',
  'July': '七月',
  'August': '八月',
  'September': '九月',
  'October': '十月',
  'November': '十一月',
  'December': '十二月'
}
```

### 2. 动态替换月份名称
在打开补卡对话框时，通过 JavaScript 查找日期选择器面板中的月份名称元素，并替换为中文：

```javascript
const setPlaceholderAndMonth = () => {
  const popper = document.querySelector('.makeup-date-picker-popper')
  if (popper) {
    // 将月份名称改为中文
    const headerLabels = popper.querySelectorAll('.el-date-picker__header-label')
    headerLabels.forEach(label => {
      const text = label.textContent.trim()
      // 匹配 "2025 December" 或 "December 2025" 格式
      for (const [enMonth, cnMonth] of Object.entries(monthMap)) {
        if (text.includes(enMonth)) {
          // 替换月份名称
          const newText = text.replace(enMonth, cnMonth)
          label.textContent = newText
          break
        }
      }
    })
  }
}
```

### 3. 使用多个延迟确保面板已渲染
由于日期选择器面板是异步渲染的，需要使用多个延迟来确保面板已完全渲染：

```javascript
setTimeout(setPlaceholderAndMonth, 100)
setTimeout(setPlaceholderAndMonth, 300)
setTimeout(setPlaceholderAndMonth, 500)
```

### 4. 监听月份切换
使用 MutationObserver 监听日期选择器面板的变化，确保在用户切换月份时也能正确显示中文月份名称：

```javascript
let makeupMonthObserver = null

setTimeout(() => {
  const popper = document.querySelector('.makeup-date-picker-popper')
  if (popper) {
    // 先断开之前的观察器
    if (makeupMonthObserver) {
      makeupMonthObserver.disconnect()
    }
    
    // 创建新的观察器
    makeupMonthObserver = new MutationObserver(() => {
      setPlaceholderAndMonth()
    })
    
    makeupMonthObserver.observe(popper, {
      childList: true,
      subtree: true,
      characterData: true
    })
  }
}, 500)
```

### 5. 清理观察器
在对话框关闭时，需要断开观察器以避免内存泄漏：

```javascript
const cancelMakeup = () => {
  makeupDialogVisible.value = false
  // 断开月份名称观察器
  if (makeupMonthObserver) {
    makeupMonthObserver.disconnect()
    makeupMonthObserver = null
  }
}
```

## 关键要点

### 1. 选择器定位
- 使用 `popper-class` 自定义类名（如 `makeup-date-picker-popper`）来精确定位日期选择器面板
- 月份名称元素的选择器为 `.el-date-picker__header-label`

### 2. 文本匹配
- 月份名称可能以不同格式出现（如 "2025 December" 或 "December 2025"）
- 使用 `includes()` 方法进行模糊匹配，而不是精确匹配

### 3. 异步渲染处理
- 日期选择器面板是异步渲染的，需要使用多个延迟来确保面板已完全渲染
- 建议使用 100ms、300ms、500ms 等多个延迟点

### 4. 动态更新
- 使用 MutationObserver 监听面板变化，确保月份切换时也能正确显示中文
- 观察器配置应包括 `childList`、`subtree`、`characterData` 选项

### 5. 资源清理
- 在对话框关闭时，必须断开观察器以避免内存泄漏
- 建议在取消和确认操作中都断开观察器

## 注意事项

- 继续保留 `:locale="zhCn"` 配置，虽然可能不完全生效，但可以处理其他文本的中文化
- 月份名称替换是兜底方案，确保在所有情况下都能正确显示中文
- 如果 Element Plus 版本更新后修复了此问题，可以考虑移除动态替换逻辑
- 观察器的清理非常重要，避免内存泄漏和性能问题

## 星期显示中文化修复

### 问题描述
Element Plus 日期选择器面板中的星期显示（Sun/Mon/Tue等）可能仍然显示为英文，即使配置了中文 locale。

### 解决方案

#### 1. 使用 CSS `::after` 伪元素替换
在全局样式（非 scoped）中，为日期选择器面板的星期表头添加中文化处理：

```css
/* 星期显示中文化 */
.makeup-date-picker-popper .el-date-table th {
  font-size: 0 !important;
  position: relative;
}

.makeup-date-picker-popper .el-date-table th:nth-child(1)::after {
  content: '日' !important;
  font-size: 13px !important;
  display: block;
}

.makeup-date-picker-popper .el-date-table th:nth-child(2)::after {
  content: '一' !important;
  font-size: 13px !important;
  display: block;
}

/* ... 依次类推，为每个星期添加对应的中文 */
```

#### 2. 关键要点
- **全局样式**：星期显示的中文化必须在全局样式（`<style>` 非 scoped）中处理，因为日期选择器面板是通过 `popper-class` 挂载到 body 上的，不在组件作用域内
- **隐藏原文本**：设置 `font-size: 0` 来隐藏原始的英文星期文本
- **伪元素替换**：使用 `::after` 伪元素的 `content` 属性来显示中文星期
- **选择器优先级**：使用 `!important` 确保样式优先级足够高，能够覆盖 Element Plus 的默认样式

#### 3. 星期映射关系
- Sun/Sunday → 日
- Mon/Monday → 一
- Tue/Tuesday → 二
- Wed/Wednesday → 三
- Thu/Thursday → 四
- Fri/Friday → 五
- Sat/Saturday → 六

## 日期选择器面板不透明设置

### 问题描述
Element Plus 日期选择器面板可能默认有透明度，导致背景不够清晰，影响视觉效果。即使设置了背景色和 `opacity: 1`，面板仍然可能显示为透明。

### 问题原因
1. **Teleport 机制**：Element Plus 的日期选择器弹出面板通过 `teleport` 渲染到 `body` 下，scoped 样式无法生效
2. **样式作用域隔离**：scoped 样式无法穿透到 teleport 渲染的元素
3. **子元素继承**：即使设置了容器背景色，子元素可能仍然有透明背景，导致整体看起来透明
4. **样式优先级**：即使使用了 `:deep()` 和 `!important`，如果选择器不够具体或未覆盖所有关键元素，仍可能被覆盖

### 解决方案

#### 1. 在全局样式中设置容器和所有关键子元素
参考 Dialog 背景透明修复经验，在全局样式（非 scoped）中，为容器和所有关键子元素明确设置背景色：

```css
<style>
/* 设置容器本身 */
.makeup-date-picker-popper {
  background-color: #ffffff !important;
  opacity: 1 !important;
}

/* 设置面板容器 */
.makeup-date-picker-popper .el-picker-panel {
  background-color: #ffffff !important;
  opacity: 1 !important;
}

/* 设置面板主体 */
.makeup-date-picker-popper .el-picker-panel__body {
  background-color: #ffffff !important;
  opacity: 1 !important;
}

/* 设置日期选择器头部 */
.makeup-date-picker-popper .el-date-picker__header {
  background-color: #ffffff !important;
}

/* 设置时间选择头部 */
.makeup-date-picker-popper .el-date-picker__time-header {
  background-color: #ffffff !important;
}

/* 设置日期表格 */
.makeup-date-picker-popper .el-date-table {
  background-color: #ffffff !important;
}

/* 表格单元格保持透明（避免覆盖选中状态） */
.makeup-date-picker-popper .el-date-table th,
.makeup-date-picker-popper .el-date-table td {
  background-color: transparent !important;
}

.makeup-date-picker-popper .el-date-table-cell {
  background-color: transparent !important;
}
</style>
```

#### 2. 在 scoped 样式中也设置（作为兜底）
在 scoped 样式中也设置，确保样式优先级：

```css
.makeup-date-picker :deep(.el-picker-panel) {
  background-color: #ffffff !important;
  opacity: 1 !important;
}

.makeup-date-picker :deep(.el-picker-panel__body) {
  background-color: #ffffff !important;
  opacity: 1 !important;
}
```

#### 3. 关键要点
- **全局样式优先**：日期选择器弹出面板通过 teleport 渲染，必须在全局样式（非 scoped）中设置
- **覆盖所有关键元素**：不仅要设置容器，还要设置所有可能影响透明度的子元素（header、time-header、date-table 等）
- **使用 background-color**：使用 `background-color` 而不是 `background`，确保样式更明确
- **使用 !important**：使用 `!important` 确保样式优先级足够高，能够覆盖 Element Plus 的默认样式
- **明确背景色**：使用 `#ffffff` 而不是 `white`，确保颜色值明确
- **设置 opacity**：明确设置 `opacity: 1` 确保完全不透明
- **表格单元格透明**：日期表格的单元格（th、td、cell）应保持透明，避免覆盖选中状态等视觉效果
