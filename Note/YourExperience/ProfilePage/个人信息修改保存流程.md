# ProfilePage - 个人信息修改保存流程

## 保存流程详细说明

### 问题
需要明确个人信息修改页面点击"保存修改"按钮后的完整执行步骤，确保流程符合接口文档要求。

### 分析
根据接口文档和代码实现，个人信息修改保存流程包含以下步骤：
1. 表单验证
2. 密码确认对话框
3. 数据提交
4. 响应处理
5. 状态更新

#### 4. 执行保存操作
```javascript
const confirmPasswordAndSave = async () => {
  if (!passwordConfirmFormRef.value) return
  
  try {
    await passwordConfirmFormRef.value.validate()
  } catch (error) {
    return
  }

  isPasswordConfirmLoading.value = true
  try {
    const token = localStorage.getItem('token')
    if (!token) {
      ElMessage.error('请先登录')
      router.push('/login')
      return
    }

    const updateData = { 
      ...formData,
      password: passwordConfirmForm.password
    }

    const response = await updateStudentInfo(token, updateData)
    
    if (response.code === 200) {
      ElMessage.success('个人信息更新成功')
      if (response.data && response.data.token) {
        localStorage.setItem('token', response.data.token)
      }
      originalData.value = { ...formData }
      isEditing.value = false
      passwordConfirmDialogVisible.value = false
      resetPasswordConfirmForm()
    } else {
      ElMessage.error(response.message || '更新个人信息失败')
    }
  } catch (error) {
    ElMessage.error('更新个人信息失败：' + error.message)
    if (error.message.includes('Token无效') || error.message.includes('请重新登录')) {
      localStorage.removeItem('token')
      localStorage.removeItem('userInfo')
      router.push('/login')
    }
  } finally {
    isPasswordConfirmLoading.value = false
  }
}
```

**位置**: `ProfilePageDesktop.vue` 第639-686行

**执行步骤**:
1. 验证密码确认表单
2. 设置加载状态为true
3. 从localStorage获取token
4. 检查token是否存在，不存在则跳转登录页
5. 构建更新数据对象（包含所有表单字段和密码）
6. 调用`updateStudentInfo`接口
7. 处理响应：
   - 成功（code === 200）：
     - 显示成功提示
     - 更新localStorage中的token（如果响应中包含新token）
     - 更新originalData为当前formData
     - 关闭编辑模式
     - 关闭密码确认对话框
     - 重置密码确认表单
   - 失败：
     - 显示错误提示
     - 如果是Token无效，清除token和userInfo，跳转登录页
8. 设置加载状态为false

#### 5. API接口调用
```javascript
export const updateStudentInfo = async (token, data) => {
  try {
    const response = await api.put('/api/v1/students/update-info', data, {
      params: { token }
    })
    return response.data
  } catch (error) {
    if (error.response) {
      const status = error.response.status
      if (status === 401) {
        throw new Error('Token无效，请重新登录')
      } else if (status === 400) {
        throw new Error('参数验证失败，请检查输入信息')
      } else if (status >= 500) {
        throw new Error('服务器错误，请稍后重试')
      } else {
        throw new Error(error.response.data?.message || '更新个人信息失败')
      }
    } else {
      throw new Error('网络错误，更新个人信息失败')
    }
  }
}
```

**位置**: `src/api/student.js` 第117-139行

### 接口文档要求

根据`接口文档.md`第805-880行，更新学生信息接口要求：

#### 请求参数验证规则
| 字段 | 验证规则 | 错误信息 |
|------|----------|----------|
| name | 不能为空 | 名字不能为空 |
| studentId | 以20-30开头的10位数字 | 学号格式不正确 |
| gender | 男或女 | 性别必须为男或女 |
| phoneNumber | 11位手机号格式（1[3456789]开头） | 手机号格式不正确 |
| college | 不能为空 | 学院不能为空 |
| major | 不能为空 | 专业不能为空 |
| grade | 1-5之间的整数 | 年级必须大于0且小于6 |
| classNum | 1-100之间的整数 | 班级必须大于0且小于101 |
| password | 6-16位字符 | 密码长度必须在6到16之间 |

#### 业务规则
1. 需要有效的JWT token进行身份验证
2. 更新成功后返回新的JWT token
3. 所有字段都需要重新验证
4. 密码会被加密存储

### 状态管理

#### 1. 编辑状态
- `isEditing`: 控制是否处于编辑模式
- 保存成功后设置为`false`，退出编辑模式

#### 2. 原始数据
- `originalData`: 保存原始数据，用于重置表单
- 保存成功后更新为当前`formData`

#### 3. 加载状态
- `isLoading`: 控制页面加载状态
- `isPasswordConfirmLoading`: 控制密码确认对话框加载状态

#### 4. 对话框状态
