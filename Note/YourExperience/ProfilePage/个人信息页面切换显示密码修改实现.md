# ProfilePage - 个人信息页面切换显示密码修改实现

## 问题描述

用户要求在个人信息页面添加一个按钮，点击后隐藏所有基本信息修改区域，显示密码修改表单，不使用对话框实现。

## 联网学习总结（8个内容）

### 1. Vue3 v-if 条件渲染最佳实践
- 使用 `v-if` 控制元素的显示和隐藏
- `v-if="!showPasswordSection"` 表示当 `showPasswordSection` 为 `false` 时显示
- `v-if="showPasswordSection"` 表示当 `showPasswordSection` 为 `true` 时显示

### 2. Vue3 响应式变量 ref 切换显示隐藏
- 使用 `ref(false)` 创建响应式变量
- 通过 `showPasswordSection.value = !showPasswordSection.value` 切换状态
- 确保变量在模板中正确绑定

### 3. Element Plus 表单切换显示 v-if vs v-show
- `v-if` 会完全移除/添加 DOM 元素，适合切换频率低的场景
- `v-show` 只是切换 CSS 的 `display` 属性，适合切换频率高的场景
- 本场景使用 `v-if` 更合适，因为切换频率低

### 4. Vue3 Composition API ref 响应式调试方法
- 在浏览器开发者工具中检查响应式变量的值
- 使用 `console.log` 输出变量值（开发环境）
- 确保变量名在模板和脚本中一致

### 5. 移动端表单切换显示实现
- 移动端和桌面端使用相同的逻辑
- 确保 `v-if` 条件在移动端和桌面端都正确添加
- 检查按钮点击事件是否正确绑定

### 6. Vue3 单页面多视图切换显示
- 使用状态变量控制不同区域的显示
- 确保同一时间只显示一个区域
- 切换时重置表单状态

### 7. Element Plus 按钮切换状态管理
- 按钮文本根据状态动态显示
- 使用 `{{ showPasswordSection ? '返回基本信息' : '修改密码' }}`
- 按钮点击事件切换状态

### 8. Vue3 表单验证切换表单清空验证状态
- 切换表单时调用 `clearValidate()` 清空验证状态
- 重置表单数据
- 确保切换流畅

## 问题修复

### 问题
Mobile 版本点击"修改密码"按钮后，基本信息区域没有隐藏。

### 原因
Mobile 版本的基本信息表单区域（`form-section`）缺少 `v-if="!showPasswordSection"` 条件，导致即使切换了状态，基本信息区域也不会隐藏。

### 修复
在 Mobile 版本的第52行添加 `v-if="!showPasswordSection"` 条件：
```vue
<div class="form-section" v-if="!showPasswordSection">
```

## 技术要点总结

1. **v-if 条件渲染**：使用 `v-if="!showPasswordSection"` 控制基本信息显示，`v-if="showPasswordSection"` 控制密码修改表单显示
2. **响应式状态管理**：使用 `ref(false)` 创建响应式变量，通过 `.value` 访问和修改
3. **按钮动态文本**：使用三元表达式根据状态动态显示按钮文本
4. **表单重置**：切换时调用 `resetPasswordForm()` 重置表单状态
5. **一致性检查**：确保 Desktop 和 Mobile 版本使用相同的逻辑和条件
