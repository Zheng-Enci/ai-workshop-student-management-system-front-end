# 头像裁剪功能实现经验

## 需求
用户上传头像时，需要能够选择要上传的图片区域（正方形裁剪框），只有裁剪框内的图片才会被上传到服务器。

## 实现方案
参考8家大厂企业级实现（微信、淘宝、京东、Ant Design、Element Plus、GitHub、企业微信、CSDN），使用Canvas API实现图片裁剪功能。

## 核心实现

### 1. 裁剪对话框
使用Element Plus Dialog组件，禁止点击遮罩和ESC键关闭，确保用户完成裁剪操作。

### 2. Canvas裁剪实现
- **图片显示**：使用Canvas绘制图片，支持拖动和缩放
- **裁剪框**：正方形裁剪框，使用绝对定位，box-shadow实现遮罩效果
- **实时预览**：裁剪框外显示半透明黑色遮罩

### 3. 交互功能
- **拖动**：鼠标拖拽移动图片位置（mousedown/mousemove/mouseup）
- **缩放**：鼠标滚轮或按钮控制（0.5-3倍）
- **重置**：恢复初始状态
- **取消**：关闭对话框

### 4. 裁剪计算
```javascript
// 1. 获取裁剪框在画布上的位置
const cropX = cropRect.left - canvasRect.left
const cropY = cropRect.top - canvasRect.top
const cropSize = cropRect.width

// 2. 计算裁剪区域在原图中的位置
const sourceX = (cropX - imageX) / scale
const sourceY = (cropY - imageY) / scale
const sourceSize = cropSize / scale

// 3. 使用Canvas提取裁剪区域
ctx.drawImage(img, sourceX, sourceY, sourceSize, sourceSize, 0, 0, cropSize, cropSize)
```

## 关键要点

- **Canvas绘制**：使用Canvas API绘制图片和遮罩
- **坐标转换**：画布坐标转换为原图坐标（考虑缩放和位置）
- **实时预览**：拖动和缩放时实时重绘画布
- **裁剪精度**：准确计算裁剪区域，确保裁剪结果正确

## 使用场景

- 头像上传
- 图片裁剪（需要选择区域）
- 图片编辑

## 移动端适配

### 触摸事件支持
- **触摸拖动**：添加touchstart/touchmove/touchend事件支持触摸拖动
- **触摸优化**：使用touch-action: manipulation和touch-action: none优化触摸响应
- **事件兼容**：同时支持鼠标事件和触摸事件，确保跨设备兼容

### 触摸捏合缩放
- **捏合手势识别**：检测两个触摸点（e.touches.length === 2）实现捏合缩放
- **距离计算**：使用getDistance函数计算两个触摸点之间的距离
- **缩放计算**：根据距离变化比例（currentDistance / initialDistance）计算缩放比例
- **中心点缩放**：以两个触摸点的中心点为基准进行缩放，保持缩放中心稳定
- **状态管理**：使用isPinching状态区分拖动和缩放操作
- **平滑过渡**：支持从捏合缩放平滑过渡到单点拖动

### 移动端尺寸适配
- **对话框宽度**：使用90%宽度适配移动端屏幕
- **裁剪区域高度**：移动端使用300px（比电脑端400px小）
- **裁剪框尺寸**：移动端最大250px（比电脑端300px小）
- **按钮尺寸**：使用small尺寸适配移动端

### 交互优化
- **使用:active替代:hover**：移动端不支持hover，使用:active提供触摸反馈
- **触摸反馈**：点击时显示scale(0.98)效果
- **触摸区域**：确保所有交互元素符合移动端最小触摸区域标准（44x44px）

## 注意事项

- Canvas API需要浏览器支持
- 裁剪框必须是正方形
- 需要处理图片加载失败的情况
- 裁剪后需要压缩再上传
- 移动端需要同时支持触摸和鼠标事件
- 移动端尺寸需要适配小屏幕
