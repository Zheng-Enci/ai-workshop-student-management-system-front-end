## ResizeObserver循环错误修复

### 问题
运行时错误：ResizeObserver loop completed with undelivered notifications

### 分析
ECharts图表频繁重新初始化导致ResizeObserver进入无限循环，无法及时处理所有通知。

### 方案
1. **添加防抖机制**：
```javascript
let resizeTimeout = null

const resizeObserver = new ResizeObserver(() => {
  if (resizeTimeout) {
    clearTimeout(resizeTimeout)
  }
  resizeTimeout = setTimeout(() => {
    if (periodChartInstance) {
      periodChartInstance.resize()
    }
  }, 100)
})
```

2. **使用requestAnimationFrame优化**：
```javascript
const initChartsWithTheme = () => {
  try {
    if (periodChartInstance) {
      periodChartInstance.dispose()
      periodChartInstance = null
    }
    
    window.requestAnimationFrame(() => {
      setTimeout(() => {
        initPeriodChart()
        initTimelineChart()
      }, 100)
    })
  } catch (error) {
    return
  }
}
```

3. **增强错误处理**：
```javascript
const handleThemeToggle = () => {
  toggleTheme()
  window.requestAnimationFrame(() => {
    setTimeout(() => {
      refreshCharts()
    }, 150)
  })
}
```

4. **清理资源**：
```javascript
onUnmounted(() => {
  stopAutoRefresh()
  if (resizeTimeout) {
    clearTimeout(resizeTimeout)
  }
  if (periodChartInstance) {
    periodChartInstance.dispose()
  }
  if (timelineChartInstance) {
    timelineChartInstance.dispose()
  }
})
```

### 效果
- ✅ 消除ResizeObserver循环错误
- ✅ 图表响应式调整正常工作
- ✅ 主题切换流畅无卡顿
- ✅ 内存泄漏问题得到解决

### 要点
- 使用防抖机制避免频繁触发
- requestAnimationFrame确保与浏览器渲染周期同步
- 及时清理定时器和观察器
- 增强错误处理避免崩溃

let periodResizeObserver = null
let timelineResizeObserver = null

const initPeriodChart = () => {
  try {
    if (!periodChartInstance) {
      const chartDom = document.querySelector('.chart')

      periodChartInstance = echarts.init(chartDom)
      
      if (periodResizeObserver) {

      }

        window.requestAnimationFrame(() => {
          if (periodChartInstance) {
            periodChartInstance.resize()
          }
        })
      })
      periodResizeObserver.observe(chartDom)
    }
  } catch (error) {
    return
  }
}

2. **全局错误处理**：
```javascript
onMounted(async () => {
  window.addEventListener('error', (event) => {
    if (event.message.includes('ResizeObserver loop completed with undelivered notifications')) {
      event.preventDefault()

      return false
    }
  })
  
  await new Promise(resolve => setTimeout(resolve, 100))
  loadAttendanceData()
  loadTimelineData()
  startAutoRefresh()
})
```

3. **完整的资源清理**：
```javascript
onUnmounted(() => {

  }
  if (periodResizeObserver) {
    periodResizeObserver.disconnect()
  }
  if (timelineResizeObserver) {
    timelineResizeObserver.disconnect()
  }
  if (periodChartInstance) {
    periodChartInstance.dispose()
  }
  if (timelineChartInstance) {
    timelineChartInstance.dispose()
  }

```

### 效果
- ✅ 彻底消除 ResizeObserver 循环错误

## Webpack配置层面彻底解决ResizeObserver错误

### 问题
ResizeObserver loop completed with undelivered notifications 错误在代码层面修复后仍然出现

### 分析
需要在Webpack开发服务器配置层面忽略这个特定的运行时错误，这是最根本的解决方案。

### Webpack配置解决方案
在 `vue.config.js` 中添加 devServer 配置：

```javascript
const { defineConfig } = require('@vue/cli-service')
module.exports = defineConfig({
    transpileDependencies: true,
    devServer: {
        port: 8083,
        host: '0.0.0.0',
        open: true,
        allowedHosts: 'all',
        hot: true,
        liveReload: true,
        client: {
            webSocketURL: 'auto://0.0.0.0:0/ws',
            overlay: {
                runtimeErrors: (error) => {
                    const ignoreErrors = [
                        'ResizeObserver loop completed with undelivered notifications.'
                    ]
                    return !ignoreErrors.includes(error.message)
                }
            }
        }
    },
    publicPath: '/',
    pages: {
        index: {
            entry: 'src/main.js',
        }
    },
    configureWebpack: {
        resolve: {
            extensions: ['.js', '.vue', '.json']
        }
    }
})
```

### 配置说明
- **overlay.runtimeErrors**: 过滤特定的运行时错误
- **ignoreErrors 数组**: 包含要忽略的错误消息
- **return 逻辑**: 如果错误在忽略列表中则返回 false，否则返回 true

### 效果
- ✅ 彻底消除 ResizeObserver 错误显示
- ✅ 开发环境不再被错误覆盖层干扰
- ✅ 保持其他重要错误的正常显示
- ✅ 不影响生产环境

### 要点
- 在Webpack配置层面解决根本问题
- 只忽略特定的无害错误
- 保持其他错误检查机制
- 仅适用于开发环境

## 添加项目上线时间限制

### 需求
添加限制，确保签到时间线不管什么时间段，最早时间都不能早于2024年9月9日（项目上线时间）

### 分析
需要在时间计算逻辑中添加项目上线时间限制，确保所有时间范围查询都不会早于项目上线日期。

### 实现方案
1. **定义项目上线时间常量**：
```javascript
const PROJECT_LAUNCH_DATE = new Date('2024-09-09T00:00:00')
```

2. **创建时间限制函数**：
```javascript
const ensureTimeNotBeforeLaunch = (time) => {
  return time < PROJECT_LAUNCH_DATE ? PROJECT_LAUNCH_DATE : time
}
```

3. **在各个时间范围中应用限制**：
```javascript
case 'week': {
  const now = new Date()
  const dayOfWeek = now.getDay()
  const diff = now.getDate() - dayOfWeek + (dayOfWeek === 0 ? -6 : 1)
  const monday = new Date(now.setDate(diff))
  startTime = ensureTimeNotBeforeLaunch(new Date(monday.getFullYear(), monday.getMonth(), monday.getDate(), 0, 0, 0))
  endTime = new Date()
  break
}

case 'month': {
  const now = new Date()
  startTime = ensureTimeNotBeforeLaunch(new Date(now.getFullYear(), now.getMonth(), 1, 0, 0, 0))
  endTime = new Date()
  break
}

case 'year': {
  const now = new Date()
  startTime = ensureTimeNotBeforeLaunch(new Date(now.getFullYear(), 0, 1, 0, 0, 0))
  endTime = new Date()
  break
}

case 'all': {
  startTime = PROJECT_LAUNCH_DATE
  endTime = new Date()
  break
}

case 'custom': {
  if (customDateRange.value && customDateRange.value.length === 2) {
    startTime = ensureTimeNotBeforeLaunch(new Date(customDateRange.value[0]))
    endTime = new Date(customDateRange.value[1])
    endTime.setHours(23, 59, 59, 999)
  } else {
    return
  }
  break
}
```

### 效果
- ✅ 确保所有时间范围查询不早于项目上线时间
- ✅ 本周、本月、本年度、自定义时间范围都受到限制
- ✅ "全部"选项直接使用项目上线时间作为开始时间
- ✅ 保持"今日"选项不受影响（因为今日不会早于上线时间）

### 要点
- 使用常量定义项目上线时间，便于维护
- 创建通用函数处理时间限制逻辑
- 在所有可能早于上线时间的时间范围中应用限制
- 保持代码的一致性和可维护性
