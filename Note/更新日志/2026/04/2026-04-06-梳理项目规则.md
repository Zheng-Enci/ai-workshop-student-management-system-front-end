# 2026-04-06

## Note/项目规则.md
- 新建项目规则文件，系统梳理项目中涉及的各类规范
- 包含代码规范、命名约定、目录结构规范、提交规范、开发流程规范等

## 代码规范
- 缩进规范：Tab 缩进，4 空格，最大行宽 120 字符
- 文件命名规范：Vue 组件帕斯卡命名、CSS 短横线命名、JS 驼峰命名
- 目录结构规范：api、assets、components、composables、config、router、stores、views
- CSS 命名规范：BEM 或短横线命名，移动端/桌面端样式分开管理
- 组件代码结构：依赖导入区、组件导入区、常量定义区、响应式状态、计算属性、方法定义、生命周期钩子

## 代码质量规范
- ESLint 规则：代码格式、代码质量、最佳实践规则配置
- Stylelint 规则：CSS 质量、现代语法、性能优化规则配置
- Prettier 配置：行宽、缩进、引号、尾随逗号等格式化配置

## Git 提交规范
- 提交信息格式：type(scope): subject，body，footer
- Type 类型定义：feat、fix、docs、style、refactor、perf、test、build、ci、chore、revert、types、wip、release
- 提交规则：type 小写、subject 不为空、不以句号结尾、字符限制
- Git Hooks：Husky + Lint-staged 自动检查

## 开发流程规范
- npm 脚本命令：dev、build、lint、format、check 等
- 代码质量检查流程：开发时自动检查、提交前全面检查、提交时自动处理

## 命名约定
- 变量命名：ref 驼峰、常量全大写、布尔 is/has/can 前缀、事件 on/handle 前缀
- 文件命名对照表：页面、样式、逻辑、配置、工具、数据加载器等
- API 文件命名：TypeScript 和 JavaScript 文件命名规范

## 技术栈规范
- 核心技术：Vue 3、Vue Router、Pinia、Element Plus、Vite、ECharts、Axios
- 开发工具：ESLint、Prettier、Stylelint、Husky、Lint-staged、Commitlint
- 浏览器兼容：>1%、最后 2 版本、非 ie 11

## 更新日志规范
- 日志文件位置：Note/更新日志/YYYY/MM/
- 日志文件命名：YYYY-MM-DD-功能描述.md
- 日志内容格式：按文件和模块分类记录变更

## 其他规范
- 图片资源命名：英文下划线分隔
- 组件引入规范：Element Plus 按需引入
- 状态管理规范：Pinia 状态管理
- 路由规范：Vue Router 4 懒加载
- 环境配置：src/config/index.js
