## 重构内容

### 整体重构
- 删除原有超级管理员页面代码，重新设计页面结构
- 采用模块化表单组件，提升代码可维护性
- 分离电脑端和移动端，适配不同设备

### 多关键词搜索
- 实现支持多个关键词的搜索功能
- 支持按学号、姓名等多种字段组合搜索

### 功能实现

#### 查看考勤记录
- 集成 ViewAttendanceRecordsForm 组件
- 支持查看学生的完整考勤记录

#### 修改学生身份
- 集成 ChangeLevelForm 组件
- 支持调整学生的身份类型

#### 修改所属管理员
- 集成 ChangeAdminForm 组件
- 支持变更学生所属的管理员

#### 补卡功能
- 集成 MakeupAttendanceForm 组件
- 支持使用特殊密码为学生补卡
- 自动获取当前特殊密码
- 自动传入选中学生信息
- 提供6个时间快捷选项（昨天/今天的上午/下午/晚上）
- 时间格式为 YYYY-MM-DDTHH:mm:ss

#### 考勤热力图
- 新增 HeatmapChartForm.vue 组件
- 支持展示学生一周七天、三个时段（上午/下午/晚上）的签到热力图
- 使用 ECharts 热力图可视化展示签到次数分布
- 支持深色/浅色主题自动适配
- 提供加载中、无数据等状态展示
- 样式类名采用 HeatmapChartForm-* 前缀，符合表单组件命名规范
- 依赖学生管理页面的考勤服务模块，支持复用现有接口
- 通过 defineExpose 暴露 openHeatmapDialog 方法供父组件调用
- 可在 AdminPageDesktop.vue 中集成，点击热力图按钮调用

#### 考勤趋势图
- 新增 AttendanceTrendChartForm.vue 组件
- 支持展示学生签到累计趋势图
- 使用 ECharts 折线图可视化展示签到次数变化
- 支持深色/浅色主题自动适配
- 提供加载中、无数据等状态展示
- 样式类名采用 AttendanceTrendChartForm-* 前缀，符合表单组件命名规范
- 依赖学生管理页面的考勤服务模块，支持复用现有接口
- 通过 defineExpose 暴露 openTrendChartDialog 方法供父组件调用
- 可在 AdminPageDesktop.vue 中集成，点击趋势图按钮调用
- 横坐标自动补全从最早到最晚考勤日期的所有日期，无考勤记录的日期签到次数为0

#### 创建积分记录
- 新增 CreatePointsRecordForm.vue 组件
- 支持通过特殊密码为指定学生创建积分记录
- 改分分数：数字输入框，支持正负值（正数为加分，负数为扣分），范围 -999 到 999
- 改分理由：多行文本输入，最长500字符，显示字数统计
- 表单验证：分数不能为空，理由不能为空且不超过500字符
- API 集成：调用 SpecialPasswordApi.createPointsRecord 创建积分记录
- 加载状态：显示全局加载蒙版
- 错误处理：友好的错误提示信息
- 在 AdminPageDesktop.vue 中集成，点击"修改积分"按钮打开表单
- 遵循现有表单组件风格和代码规范
- 使用特殊密码通过 Header X-Special-Password 传递

#### 查看改分记录
- 新增 ViewPointsRecordsForm.vue 组件
- 支持查看指定学生的所有改分记录
- 使用表格展示改分记录，包含字段：时间、分数、理由
- 积分变更显示：使用 Tag 组件，加分显示绿色标签，扣分显示红色标签
- 时间格式化：将创建时间格式化为 YYYY-MM-DD HH:mm:ss
- 无数据提示：当没有改分记录时显示友好的空状态提示
- API 集成：调用 PointsApi.getAllAdjustRecordsByStudentInfoId 获取改分记录
- 加载状态：显示全局加载蒙版
- 错误处理：友好的错误提示信息
- 在 AdminPageDesktop.vue 中集成，点击"改分记录"按钮打开表单
- 遵循现有表单组件风格和代码规范
- 长文本处理：理由列使用 el-popover 组件展示，鼠标悬停显示完整内容，支持自动换行和长文本显示
- 主题适配：支持深色/浅色主题，日间模式使用白色背景+黑色文字，夜间模式使用黑色背景+白色文字，确保高对比度可读性
- 使用 popper-class 自定义样式，通过 !important 强制覆盖全局样式，确保文字颜色正确显示

#### 修改学生信息
- 新增 UpdateStudentInfoForm.vue 组件
- 支持修改学生的基本信息，包含字段：姓名、学号、性别、手机号、学院、专业、年级、班级、密码
- 学号验证：10位数字，前两位为年份，支持20-30开头的学号（即20xxxxxxxx到30xxxxxxxx）
- 密码字段设计：
  - 密码为可选字段，不填写则不修改学生原密码
  - 填写密码时必须验证长度为6-20位
  - 提交时只有当密码字段有值才在请求体中包含该字段
  - 提示文字明确告知用户"请输入新密码（6-20位），不填写则不修改密码"
- 表单验证：
  - 姓名：必填，长度2-20字符
  - 学号：必填，10位数字且以20-30开头
  - 性别：必填，选项为男/女
  - 手机号：必填，11位手机号格式（1[3456789]开头）
  - 学院：必填，最大50字符
  - 专业：必填，最大50字符
  - 年级：必填，1-5之间的整数
  - 班级：必填，1-100之间的整数
  - 密码：可选，有值时验证长度6-20位
- API 集成：调用 SpecialPasswordApi.updateStudentWithSpecialPassword 修改学生信息
- 特殊密码验证：通过 Header 的 X-Special-Password 和 URL 参数的 student-id 传递验证信息
- 加载状态：显示全局加载蒙版
- 错误处理：友好的错误提示信息
- 在 AdminPageDesktop.vue 中集成，点击"修改信息"按钮打开表单
- 遵循现有表单组件风格和代码规范
- 数据安全：密码字段在表单打开时清空，不显示学生原密码，确保密码安全
