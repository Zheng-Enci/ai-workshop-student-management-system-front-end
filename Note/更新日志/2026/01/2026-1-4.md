## 🔧 十二、依赖检查工具优化

### ✨ depcheck 插件智能过滤

#### 优化内容
- **文件**: `vite-plugin-depcheck.js`
- **修改**: 新增智能过滤功能，自动过滤已配置忽略的依赖
- **目的**: 只显示真正未使用的依赖，避免误报

#### 新增功能

**1. 智能依赖过滤**
- 自动读取 `.depcheckrc.json` 配置文件
- 收集所有需要忽略的依赖（从 `ignoreDeps`、`ignore`、`ignoreMatches` 字段）
- 解析 depcheck 的输出结果
- 过滤掉已配置忽略的依赖
- 只显示真正未使用的依赖

**2. 过滤逻辑**
- 支持 `ignoreDeps` 数组：直接忽略的依赖列表
- 支持 `ignore` 数组：兼容旧配置格式
- 支持 `ignoreMatches` 模式匹配：使用通配符匹配依赖（如 `@babel/*`、`eslint-*`）
- 自动从 `package.json` 获取所有依赖，应用模式匹配规则

**3. 输出优化**
- 如果所有依赖都被忽略，显示 "✅ 未发现未使用的依赖"
- 如果有真正未使用的依赖，显示警告信息
- 避免显示已配置忽略的依赖，减少干扰

#### 优化效果

**优化前**:
```
⚠️  发现依赖问题:
Unused dependencies
* vue
* vue-router
* element-plus
... (所有已配置忽略的依赖)
```

**优化后**:
```
✅ 未发现未使用的依赖
```
或
```
⚠️  发现依赖问题:
Unused dependencies
* 真正未使用的依赖1
* 真正未使用的依赖2
```

#### 技术实现

- 使用 `readFileSync` 读取配置文件
- 使用正则表达式匹配 `ignoreMatches` 模式
- 解析 depcheck 的输出格式，逐行过滤
- 保持原有的错误处理机制

#### 配置说明

所有需要忽略的依赖已在 `.depcheckrc.json` 中配置：
- **运行时依赖**: vue、vue-router、pinia、element-plus、axios、echarts 等
- **开发依赖**: vite、eslint、prettier、stylelint、husky 等
- **模式匹配**: `@babel/*`、`@typescript-eslint/*`、`eslint-*` 等

这些依赖虽然被 depcheck 标记为"未使用"，但实际上都在使用中，只是 depcheck 无法正确检测到。
