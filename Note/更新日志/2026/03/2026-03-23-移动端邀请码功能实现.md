# 移动端邀请码功能实现

## 日期
2026-03-23

## 功能概述
在移动端导航页面中实现邀请码功能，包括邀请码卡片和邀请码表单对话框。

## 主要改动

### 1. 创建移动端邀请码表单组件
- 文件：`src/views/NavigationPage/form/mobile/InvitationCodeForm.vue`
- 功能：
  - 显示当前用户的邀请码
  - 支持复制邀请码到剪贴板
  - 支持刷新邀请码
  - 显示加载状态、错误提示和无数据提示
  - 遵循 AdminPage 表单风格

### 2. 移动端适配优化
- Dialog 宽度：从桌面端的 500px 改为 90%，适应移动端屏幕
- 邀请码字体：从 24px 改为 20px，更符合移动端显示
- 邀请码最大宽度：从 400px 改为 90%，自适应屏幕宽度
- 提示文本最大宽度：从 300px 改为 280px
- 内边距优化：加载状态、错误提示、无数据提示增加了左右 20px 的间距
- 按钮布局：添加 flex-wrap 和 justify-content: center，适应小屏幕

### 3. 在移动端导航页面中添加邀请码卡片
- 文件：`src/views/NavigationPage/NavigationPageMobile.vue`
- 在"个人中心"分组中添加"邀请码"卡片
- 点击卡片打开邀请码表单对话框
- 导入 InvitationCodeForm 组件
- 添加 invitationFormVisible ref 控制表单显示
- 实现 goToInvitationCode 函数打开表单

### 4. 修复复制功能
- 在桌面端和移动端的邀请码表单中都添加了复制功能的降级方案
- 优先使用现代 API：navigator.clipboard.writeText
- 降级方案：使用 document.execCommand('copy') 方法
  - 创建隐藏的 textarea 元素
  - 将邀请码写入 textarea
  - 选中 textarea 内容
  - 执行复制命令
  - 删除 textarea 元素
- 确保在各种环境下复制功能都能正常工作

## 技术细节

### 组件特性
- 使用 TypeScript 类型注解
- 遵循 AdminPage 表单风格
- 使用 Element Plus 组件（Dialog、Button、Icon）
- 支持暗色模式
- 错误信息在表单中直接显示，不使用 Element Message
- 复制功能具有跨浏览器兼容性

### 样式管理
- 所有样式独立在组件的 style 标签中
- 使用 flex 布局
- 响应式设计
- 支持暗色模式适配

### 复制功能兼容性
- 现代浏览器：使用 Clipboard API
- 旧版浏览器：使用 document.execCommand('copy')
- 确保在所有环境下都能正常复制

## API 调用
- 使用 `InvitationApi.getInvitationCode(token)` 获取邀请码
- Token 从 `userStore` 中获取
- 错误处理：400 状态码和通用错误

## 用户体验优化
- 页面加载时不自动获取邀请码
- 点击"邀请码"卡片时打开表单对话框
- 表单打开时自动加载邀请码
- 复制成功后显示"已复制"状态（2秒后恢复）
- 错误信息直接在表单中显示，避免弹窗干扰
- 复制功能在所有浏览器环境下都能正常工作
