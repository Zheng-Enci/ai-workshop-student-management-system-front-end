# 邀请码表单实现

## 日期
2026-03-23

## 功能概述
在导航页面中实现邀请码功能，包括邀请码卡片和邀请码表单对话框。

## 主要改动

### 1. 创建邀请码表单组件
- 文件：`src/views/NavigationPage/form/desktop/InvitationCodeForm.vue`
- 功能：
  - 显示当前用户的邀请码
  - 支持复制邀请码到剪贴板
  - 支持刷新邀请码
  - 显示加载状态、错误提示和无数据提示
  - 遵循 AdminPage 表单风格

### 2. 在导航页面中添加邀请码卡片
- 文件：`src/views/NavigationPage/NavigationPageDesktop.vue`
- 在"个人中心"分组中添加"邀请码"卡片
- 点击卡片打开邀请码表单对话框

### 3. 集成邀请码表单
- 导入 `InvitationCodeForm` 组件
- 添加 `invitationFormVisible` ref 控制表单显示
- 实现 `handleInvitationCode` 函数打开表单

### 4. 移除积分卡片下方的邀请码显示
- 删除了 `invitation-section` 块及其相关状态管理
- 清理了 `invitationManager`、`handleFetchInvitationCode`、`handleCopyInvitationCode` 等代码
- 删除了 `useInvitationCode` 的导入
- 删除了 `InvitationCode.css` 的导入

### 5. 创建邀请码样式文件
- 文件：`src/views/NavigationPage/css/InvitationCode.css`
- 包含邀请码区域的所有样式
- 所有样式名添加 `invitation-` 前缀

## 技术细节

### 组件特性
- 使用 TypeScript 类型注解
- 遵循 AdminPage 表单风格
- 使用 Element Plus 组件（Dialog、Button、Icon）
- 支持暗色模式
- 错误信息在表单中直接显示，不使用 Element Message

### 样式管理
- 所有样式独立在 `InvitationCode.css` 文件中
- 使用 flex 布局
- 响应式设计
- 支持暗色模式适配

## API 调用
- 使用 `InvitationApi.getInvitationCode(token)` 获取邀请码
- Token 从 `userStore` 中获取
- 错误处理：400 状态码和通用错误

## 用户体验优化
- 页面加载时不自动获取邀请码
- 点击"邀请码"卡片时打开表单对话框
- 表单打开时自动加载邀请码
- 复制成功后显示"已复制"状态（2秒后恢复）
- 错误信息直接在表单中显示，避免弹窗干扰
