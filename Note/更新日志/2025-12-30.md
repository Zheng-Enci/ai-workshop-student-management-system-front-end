## 添加学生签到记录缓存机制

### 修改内容
- 文件：`src/views/StudentManagerPage/js/StudentManagerPageStudentAttendanceServer.js`
- 新增静态字典用于存储已获取的学生签到记录

### 技术实现
1. **缓存结构**
   - 使用静态字典按学生ID存储签到记录
   - 键为学生ID，值为该学生的签到记录列表

2. **缓存逻辑**
   - 首次请求时从后端API获取数据并存入缓存
   - 后续请求先检查缓存，命中则直接返回
   - 避免重复请求同一学生的签到数据

### 性能优化
- 减少重复的网络请求，降低服务器压力
- 提升页面加载速度和响应性能
- 优化用户体验，特别是在查看考勤记录、热力图、趋势图时
- 减少数据传输量，提高系统整体效率

### 影响范围
- 学生考勤记录查看功能
- 学生签到热力图展示
- 学生签到趋势图展示
- 任何需要多次查看同一学生签到记录的场景

## 考勤记录对话框逻辑模块化重构

### 变更统计
- 新增文件：`src/views/StudentManagerPage/js/StudentManagerPage-Attendance_Records_Dialog.js`（133行）
- 修改文件：`src/views/StudentManagerPage/StudentManagerPageDesktop.vue`（94行新增，170行删除）

### 新增文件内容
创建独立的考勤记录对话框管理类 `StudentManagerPageAttendance_Records_Dialog`：

#### 核心功能
- `openAttendanceRecordsDialog(student)` - 打开对话框并加载学生考勤记录
- `closeAttendanceRecordsDialog()` - 关闭对话框，清理状态
- `setStudentAttendanceServer()` - 注入学生考勤服务实例

#### 响应式状态管理
```javascript
{
  attendanceRecordsDialogVisible: false,
  selectedStudent: null,
  studentAttendanceRecords: [],
  calendarValue: new Date(),
  attendanceRecordsLoading: false
}
```

#### 访问方法
- 获取/设置对话框可见性
- 获取选中学生、考勤记录、日历值
- 加载状态管理

### 主组件修改内容

#### 1. 模板层变更
- **按钮点击事件修改**：
  - 考勤记录按钮：`openAttendanceRecordsDialog(student)` → `StudentManagerPageAttendance_Records_Dialog.openAttendanceRecordsDialog(student)`
  - 补卡按钮：`openMakeupDialog(student)` → `openDatePicker(student)`
- **补卡对话框简化**：移除约70行的 `el-dialog` 完整对话框结构，替换为隐藏的 `el-date-picker` 组件

#### 2. JavaScript 逻辑变更
- `closeTrendChartDialog()` - 清理数据改为访问外部模块状态
- 日历导航函数（`prevMonth`、`nextMonth`、`goToday`）- 日历值改为访问外部模块
- 页面回退事件处理（`pageshow`）- 统一调用 `StudentManagerPageAttendance_Records_Dialog.closeAttendanceRecordsDialog()`
- 图表数据生成（`generateLineData`、`generateHeatmapData`）- 从外部模块读取考勤记录
- watch 监听器 - 监听目标改为外部模块的 `state.studentAttendanceRecords`

### 重构优势
- **职责分离**：考勤记录逻辑抽离到独立类，主组件复杂度降低
- **代码复用**：对话框状态管理集中化，便于其他组件使用
- **易于维护**：状态和操作方法封装清晰，便于后续修改和测试
- **UI 简化**：补卡功能使用隐藏日期选择器替代完整对话框，减少模板代码

### 影响范围
- 考勤记录对话框相关功能
- 学生考勤记录查看
- 签到热力图和趋势图展示
- 日历导航功能
- 补卡日期选择功能
