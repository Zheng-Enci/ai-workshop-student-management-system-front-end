# 学生管理系统接口文档

## 项目概述

本项目是一个基于Spring Boot的学生管理系统后端服务，提供学生注册、登录、签到等基础功能，以及丰富的统计分析功能。系统使用JWT进行身份认证，Redis进行缓存管理，支持分时段签到功能，并提供多维度的数据统计和分析能力。

### 主要功能模块

1. **学生管理模块**
   - 学生注册和登录
   - Token验证
   - 年级、专业、总数统计

2. **考勤管理模块**
   - 分时段签到功能
   - 优秀学生统计
   - 月度/日度考勤统计

3. **数据统计模块**
   - 学生分布统计（按年级、专业）
   - 考勤数据分析
   - 实时统计报表

### 系统架构

系统采用分层架构设计，遵循Controller-Service-Repository模式：

1. **表现层 (Controller)**: 处理HTTP请求，参数验证，响应格式化
2. **业务层 (Service)**: 核心业务逻辑处理，事务管理
3. **数据访问层 (Repository)**: 数据库操作，ORM映射
4. **工具层 (Util)**: JWT工具类，通用工具方法
5. **配置层 (Config)**: 系统配置，安全配置，跨域配置

## 技术栈

- **框架**: Spring Boot 3.5.5
- **Java版本**: Java 24
- **数据库**: MySQL 8.0 (通过JPA/Hibernate)
- **缓存**: Redis 6.0+
- **认证**: Spring Security + JWT (JSON Web Token)
- **文档**: Knife4j 4.6.0
- **日志**: SLF4J + Log4j2
- **参数校验**: Spring Boot Validation

## 通用响应格式

所有接口都使用统一的响应格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200表示成功，其他表示失败 |
| message | String | 响应消息 |
| data | Object | 响应数据，具体结构根据接口而定 |

## API接口详情

### 1. 学生管理接口

#### 1.1 学生注册

**接口地址**: `POST /api/v1/students`

**请求头**:
```
Content-Type: application/json
```

**请求参数**:

```json
{
  "name": "张三",
  "studentId": "2021001001",
  "gender": "男",
  "phoneNumber": "13812345678",
  "college": "计算机学院",
  "major": "软件工程",
  "grade": 3,
  "classNum": 1,
  "password": "123456"
}
```

**参数验证规则**:

| 字段 | 验证规则 | 错误信息 |
|------|----------|----------|
| name | 不能为空 | 名字不能为空 |
| studentId | 10位数字 | 学号格式不正确 |
| gender | 男或女 | 性别必须为男或女 |
| phoneNumber | 11位手机号格式 | 手机号格式不正确 |
| college | 不能为空 | 学院不能为空 |
| major | 不能为空 | 专业不能为空 |
| grade | 1-5之间的整数 | 年级必须大于0且小于6 |
| classNum | 1-100之间的整数 | 班级必须大于0且小于101 |
| password | 6-16位字符 | 密码长度必须在6到16之间 |

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9..."
  }
}
```

失败响应 (400):
```json
{
  "code": 400,
  "message": "学号已存在",
  "data": null
}
```

或

```json
{
  "code": 400,
  "message": "手机号已存在",
  "data": null
}
```

或

```json
{
  "code": 400,
  "message": "参数验证失败",
  "data": null
}
```

**业务规则**:
- 学号必须唯一
- 手机号必须唯一
- 密码会被加密存储
- 注册成功后自动生成JWT token

## 错误码说明

| 错误码 | 说明 | 常见场景 |
|--------|------|----------|
| 200 | 操作成功 | 请求正常处理 |
| 400 | 请求参数错误或业务逻辑错误 | 参数验证失败、业务规则违反 |
| 401 | 认证失败（Token无效） | Token过期、格式错误、签名无效 |
| 403 | 权限不足 | 访问受限资源 |
| 404 | 资源不存在 | 学号不存在、接口路径错误 |
| 409 | 资源冲突 | 学号已存在、手机号已存在 |
| 500 | 服务器内部错误 | 数据库连接失败、系统异常 |

## 认证机制

### JWT Token

系统使用JWT进行身份认证，Token包含以下信息：

- **studentId**: 学号
- **traceId**: 追踪ID（UUID）
- **issuedAt**: 签发时间
- **expiration**: 过期时间（7天有效期）

### Token使用方式

1. 注册或登录成功后，系统会返回JWT token
2. 后续请求需要在Query参数中携带token
3. Token有过期时间，过期后需要重新登录

## 缓存机制

### Redis缓存

系统使用Redis缓存签到状态，防止重复签到：

- **缓存Key格式**: `attendance:{studentId}:{period}:{date}`
- **缓存值**: `true`（表示已签到）
- **过期时间**: 当天结束
- **性能指标**: 响应时间优化至100ms内

### 缓存策略

1. **签到状态缓存**: 防止同一时间段重复签到
2. **数据一致性**: 缓存与数据库同步更新
3. **缓存清理**: 每日凌晨自动清理过期缓存

## 数据库设计

### 学生表 (student_info)

```sql
CREATE TABLE student_info (
    id INT PRIMARY KEY AUTO_INCREMENT,
    student_id BIGINT UNIQUE NOT NULL,
    name VARCHAR(255) NOT NULL,
    gender VARCHAR(10) NOT NULL,
    phone_number VARCHAR(20) UNIQUE NOT NULL,
    college VARCHAR(255) NOT NULL,
    grade INT NOT NULL,
    class_num INT NOT NULL,
    password VARCHAR(255) NOT NULL,
    major VARCHAR(255) NOT NULL
);
```

### 考勤表 (attendance)

```sql
CREATE TABLE attendance (
    id INT PRIMARY KEY AUTO_INCREMENT,
    student_info_id INT NOT NULL,
    attendance_time DATETIME NOT NULL,
    FOREIGN KEY (student_info_id) REFERENCES student_info(id)
);
```
