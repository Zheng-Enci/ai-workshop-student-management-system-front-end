# 学生管理系统接口文档

## 项目概述

本项目是一个基于Spring Boot的学生管理系统后端服务，提供学生注册、登录、签到等功能。系统使用JWT进行身份认证，Redis进行缓存管理，支持分时段签到功能。

## 技术栈

- **框架**: Spring Boot 3.x
- **数据库**: MySQL (通过JPA/Hibernate)
- **缓存**: Redis
- **认证**: JWT (JSON Web Token)
- **文档**: Knife4j (Swagger)
- **日志**: Log4j2

## 基础信息

- **基础URL**: `http://10.0.48.220:7001`
- **API版本**: v1
- **数据格式**: JSON
- **字符编码**: UTF-8

## 通用响应格式

所有接口都使用统一的响应格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200表示成功，其他表示失败 |
| message | String | 响应消息 |
| data | Object | 响应数据，具体结构根据接口而定 |

## 数据模型

### 学生实体 (Student)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | 主键，自增 | 数据库主键 |
| studentId | Long | 唯一，非空 | 学号（10位数字） |
| name | String | 非空 | 学生姓名 |
| gender | String | 非空 | 性别（男/女） |
| phoneNumber | String | 唯一，非空 | 手机号 |
| college | String | 非空 | 学院 |
| grade | Integer | 非空 | 年级（1-5） |
| classNum | Integer | 非空 | 班级（1-100） |
| password | String | 非空 | 密码（加密存储） |
| major | String | 非空 | 专业 |

### 考勤实体 (Attendance)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | 主键，自增 | 数据库主键 |
| student_info_id | Integer | 非空 | 学生ID（关联student表） |
| attendance_time | LocalDateTime | 非空 | 签到时间 |

## API接口

### 1. 学生管理接口

#### 1.1 学生注册

**接口地址**: `POST /api/v1/students`

**请求头**:
```
Content-Type: application/json
```

**请求参数**:

```json
{
  "name": "张三",
  "studentId": "2021001001",
  "gender": "男",
  "phoneNumber": "13812345678",
  "college": "计算机学院",
  "major": "软件工程",
  "grade": 3,
  "classNum": 1,
  "password": "123456"
}
```

**参数验证规则**:

| 字段 | 验证规则 | 错误信息 |
|------|----------|----------|
| name | 不能为空 | 名字不能为空 |
| studentId | 10位数字 | 学号格式不正确 |
| gender | 男或女 | 性别必须为男或女 |
| phoneNumber | 11位手机号格式 | 手机号格式不正确 |
| college | 不能为空 | 学院不能为空 |
| major | 不能为空 | 专业不能为空 |
| grade | 1-5之间的整数 | 年级必须大于0且小于6 |
| classNum | 1-100之间的整数 | 班级必须大于0且小于101 |
| password | 6-16位字符 | 密码长度必须在6到16之间 |

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9..."
  }
}
```

失败响应 (400):
```json
{
  "code": 400,
  "message": "学号已存在",
  "data": null
}
```

**业务规则**:
- 学号必须唯一
- 手机号必须唯一
- 密码会被加密存储
- 注册成功后自动生成JWT token

### 2. 考勤管理接口

#### 2.1 学生签到

**接口地址**: `POST /api/v1/attendance`

**请求参数**:

| 参数名 | 类型 | 位置 | 必填 | 说明 |
|--------|------|------|------|------|
| token | String | Query | 是 | JWT token |

**签到时间段**:

| 时间段 | 时间范围 | 说明 |
|--------|----------|------|
| 上午 | 08:00 - 11:00 | 上午签到时间段 |
| 下午 | 14:00 - 17:00 | 下午签到时间段 |
| 晚上 | 19:00 - 23:58 | 晚上签到时间段 |

**响应示例**:

成功响应 (200):
```json
{
  "code": 200,
  "message": "签到成功",
  "data": {
    "message": "签到成功"
  }
}
```

失败响应 (400):
```json
{
  "code": 400,
  "message": "当前时间不在签到时间段内",
  "data": null
}
```

或

```json
{
  "code": 400,
  "message": "您已于本期签到，请勿重复签到",
  "data": null
}
```

或

```json
{
  "code": 400,
  "message": "该学号不存在",
  "data": null
}
```

**业务规则**:
- 只能在指定时间段内签到
- 每个时间段只能签到一次
- 使用Redis缓存防止重复签到
- 签到记录会保存到数据库

## 错误码说明

| 错误码 | 说明 |
|--------|------|
| 200 | 操作成功 |
| 400 | 请求参数错误或业务逻辑错误 |
| 401 | 认证失败（Token无效） |
| 500 | 服务器内部错误 |

## 认证机制

### JWT Token

系统使用JWT进行身份认证，Token包含以下信息：

- **studentId**: 学号
- **traceId**: 追踪ID（UUID）
- **issuedAt**: 签发时间
- **expiration**: 过期时间

### Token使用方式

1. 注册或登录成功后，系统会返回JWT token
2. 后续请求需要在Query参数中携带token
3. Token有过期时间，过期后需要重新登录

## 缓存机制

### Redis缓存

系统使用Redis缓存签到状态，防止重复签到：

- **缓存Key格式**: `attendance:{studentId}:{period}:{date}`
- **缓存值**: `true`（表示已签到）
- **过期时间**: 当天结束

## 数据库设计

### 学生表 (student_info)

```sql
CREATE TABLE student_info (
    id INT PRIMARY KEY AUTO_INCREMENT,
    student_id BIGINT UNIQUE NOT NULL,
    name VARCHAR(255) NOT NULL,
    gender VARCHAR(10) NOT NULL,
    phone_number VARCHAR(20) UNIQUE NOT NULL,
    college VARCHAR(255) NOT NULL,
    grade INT NOT NULL,
    class_num INT NOT NULL,
    password VARCHAR(255) NOT NULL,
    major VARCHAR(255) NOT NULL
);
```

### 考勤表 (attendance)

```sql
CREATE TABLE attendance (
    id INT PRIMARY KEY AUTO_INCREMENT,
    student_info_id INT NOT NULL,
    attendance_time DATETIME NOT NULL,
    FOREIGN KEY (student_info_id) REFERENCES student_info(id)
);
```

## 部署说明

### 环境要求

- Java 17+
- MySQL 8.0+
- Redis 6.0+
- Maven 3.6+

### 配置文件

`application.properties`:

```properties
# 数据库配置
spring.datasource.url=jdbc:mysql://localhost:3306/student_management
spring.datasource.username=root
spring.datasource.password=password

# JWT配置
jwt.secret=your-secret-key-here
jwt.expiration=86400000

# Redis配置
spring.redis.host=localhost
spring.redis.port=6379
spring.redis.password=
```

### 启动步骤

1. 启动MySQL数据库
2. 启动Redis服务
3. 执行数据库脚本创建表结构
4. 修改配置文件中的数据库连接信息
5. 运行 `mvn spring-boot:run` 启动应用

## 注意事项

1. **时间限制**: 签到功能有严格的时间段限制，请在规定时间内测试
2. **数据唯一性**: 学号和手机号必须唯一，重复注册会失败
3. **Token安全**: JWT token包含敏感信息，请妥善保管
4. **密码安全**: 密码在数据库中加密存储，无法直接查看
5. **缓存清理**: Redis缓存会在每天结束时自动清理
